_metadata_requests.py 
1    """ 
2    Metadata Routing Utility 
3     
4    In order to better understand the components implemented in this file, one 
5    needs to understand their relationship to one another. 
6     
7    The only relevant public API for end users are the ``set_{method}_request`` methods, 
8    e.g. ``estimator.set_fit_request(sample_weight=True)``. However, third-party 
9    developers and users who implement custom meta-estimators, need to deal with 
10   the objects implemented in this file. 
11    
12   All estimators (should) implement a ``get_metadata_routing`` method, returning 
13   the routing requests set for the estimator. This method is automatically 
14   implemented via ``BaseEstimator`` for all simple estimators, but needs a custom 
15   implementation for meta-estimators. 
16    
17   In non-routing consumers, i.e. the simplest case, e.g. ``SVM``, 
18   ``get_metadata_routing`` returns a ``MetadataRequest`` object. 
19    
20   In routers, e.g. meta-estimators and a multi metric scorer, 
21   ``get_metadata_routing`` returns a ``MetadataRouter`` object. 
22    
23   An object which is both a router and a consumer, e.g. a meta-estimator which 
24   consumes ``sample_weight`` and routes ``sample_weight`` to its sub-estimators, 
25   routing information includes both information about the object itself (added 
26   via ``MetadataRouter.add_self_request``), as well as the routing information 
27   for its sub-estimators. 
28    
29   A ``MetadataRequest`` instance includes one ``MethodMetadataRequest`` per 
30   method in ``METHODS``, which includes ``fit``, ``score``, etc. 
31    
32   Request values are added to the routing mechanism by adding them to 
33   ``MethodMetadataRequest`` instances, e.g. 
34   ``metadatarequest.fit.add(param="sample_weight", alias="my_weights")``. This is 
35   used in ``set_{method}_request`` which are automatically generated, so users 
36   and developers almost never need to directly call methods on a 
37   ``MethodMetadataRequest``. 
38    
39   The ``alias`` above in the ``add`` method has to be either a string (an alias), 
40   or a {True (requested), False (unrequested), None (error if passed)}``. There 
41   are some other special values such as ``UNUSED`` and ``WARN`` which are used 
42   for purposes such as warning of removing a metadata in a child class, but not 
43   used by the end users. 
44    
45   ``MetadataRouter`` includes information about sub-objects' routing and how 
46   methods are mapped together. For instance, the information about which methods 
47   of a sub-estimator are called in which methods of the meta-estimator are all 
48   stored here. Conceptually, this information looks like: 
49    
50   ``` 
51   { 
52       "sub_estimator1": ( 
53           mapping=[(caller="fit", callee="transform"), ...], 
54           router=MetadataRequest(...),  # or another MetadataRouter 
55       ), 
56       ... 
57   } 
58   ``` 
59    
60   To give the above representation some structure, we use the following objects: 
61    
62   - ``(caller=..., callee=...)`` is a namedtuple called ``MethodPair`` 
63    
64   - The list of ``MethodPair`` stored in the ``mapping`` field of a `RouterMappingPair` is 
65     a ``MethodMapping`` object 
66    
67   - ``(mapping=..., router=...)`` is a namedtuple called ``RouterMappingPair`` 
68    
69   The ``set_{method}_request`` methods are dynamically generated for estimators 
70   which inherit from the ``BaseEstimator``. This is done by attaching instances 
71   of the ``RequestMethod`` descriptor to classes, which is done in the 
72   ``_MetadataRequester`` class, and ``BaseEstimator`` inherits from this mixin. 
73   This mixin also implements the ``get_metadata_routing``, which meta-estimators 
74   need to override, but it works for simple consumers as is. 
75   """

133  def _raise_for_params(params, owner, method):
134      """Raise an error if metadata routing is not enabled and params are passed. 
135   
136      .. versionadded:: 1.4 
137   
138      Parameters 
139      ---------- 
140      params : dict 
141          The metadata passed to a method. 
142   
143      owner : object 
144          The object to which the method belongs. 
145   
146      method : str 
147          The name of the method, e.g. "fit". 
148   
149      Raises 
150      ------ 
151      ValueError 
152          If metadata routing is not enabled and params are passed. 
153      """
154      caller = (
155          f"{owner.__class__.__name__}.{method}" if method else owner.__class__.__name__
156      )
157      if not _routing_enabled() and params:
158          raise ValueError(
159              f"Passing extra keyword arguments to {caller} is only supported if"
160              " enable_metadata_routing=True, which you can set using"
161              " `sklearn.set_config`. See the User Guide"
162              " <https://scikit-learn.org/stable/metadata_routing.html> for more"
163              f" details. Extra parameters passed are: {set(params)}"
164          )
165

167  def _raise_for_unsupported_routing(obj, method, **kwargs):
168      """Raise when metadata routing is enabled and metadata is passed. 
169   
170      This is used in meta-estimators which have not implemented metadata routing 
171      to prevent silent bugs. There is no need to use this function if the 
172      meta-estimator is not accepting any metadata, especially in `fit`, since 
173      if a meta-estimator accepts any metadata, they would do that in `fit` as 
174      well. 
175   
176      Parameters 
177      ---------- 
178      obj : estimator 
179          The estimator for which we're raising the error. 
180   
181      method : str 
182          The method where the error is raised. 
183   
184      **kwargs : dict 
185          The metadata passed to the method. 
186      """
187      kwargs = {key: value for key, value in kwargs.items() if value is not None}
188      if _routing_enabled() and kwargs:
189          cls_name = obj.__class__.__name__
190          raise NotImplementedError(
191              f"{cls_name}.{method} cannot accept given metadata ({set(kwargs.keys())})"
192              f" since metadata routing is not yet implemented for {cls_name}."
193          )

215  # Request values
216  # ==============
217  # Each request value needs to be one of the following values, or an alias.
218  
219  # this is used in `__metadata_request__*` attributes to indicate that a
220  # metadata is not present even though it may be present in the
221  # corresponding method's signature.
222  UNUSED = "$UNUSED$"
223  
224  # this is used whenever a default value is changed, and therefore the user
225  # should explicitly set the value, otherwise a warning is shown. An example
226  # is when a meta-estimator is only a router, but then becomes also a
227  # consumer in a new release.
228  WARN = "$WARN$"
229  
230  # this is the default used in `set_{method}_request` methods to indicate no
231  # change requested by the user.
232  UNCHANGED = "$UNCHANGED$"
233  
234  VALID_REQUEST_VALUES = [False, True, None, UNUSED, WARN]
235  
236  
237  def request_is_alias(item):
238      """Check if an item is a valid alias. 
239   
240      Values in ``VALID_REQUEST_VALUES`` are not considered aliases in this 
241      context. Only a string which is a valid identifier is. 
242   
243      Parameters 
244      ---------- 
245      item : object 
246          The given item to be checked if it can be an alias. 
247   
248      Returns 
249      ------- 
250      result : bool 
251          Whether the given item is a valid alias. 
252      """
253      if item in VALID_REQUEST_VALUES:
254          return False
255  
256      # item is only an alias if it's a valid identifier
257      return isinstance(item, str) and item.isidentifier()
258  
259  
260  def request_is_valid(item):
261      """Check if an item is a valid request value (and not an alias). 
262   
263      Parameters 
264      ---------- 
265      item : object 
266          The given item to be checked. 
267   
268      Returns 
269      ------- 
270      result : bool 
271          Whether the given item is valid. 
272      """
273      return item in VALID_REQUEST_VALUES

282  class MethodMetadataRequest:
283      """A prescription of how metadata is to be passed to a single method. 
284   
285      Refer to :class:`MetadataRequest` for how this class is used. 
286   
287      .. versionadded:: 1.3 
288   
289      Parameters 
290      ---------- 
291      owner : str 
292          A display name for the object owning these requests. 
293   
294      method : str 
295          The name of the method to which these requests belong. 
296   
297      requests : dict of {str: bool, None or str}, default=None 
298          The initial requests for this method. 
299      """
300  
301      def __init__(self, owner, method, requests=None):
302          self._requests = requests or dict()
303          self.owner = owner
304          self.method = method
305  
306      @property
307      def requests(self):
308          """Dictionary of the form: ``{key: alias}``."""
309          return self._requests
310  
311      def add_request(
312          self,
313          *,
314          param,
315          alias,
316      ):
317          """Add request info for a metadata. 
318   
319          Parameters 
320          ---------- 
321          param : str 
322              The property for which a request is set. 
323   
324          alias : str, or {True, False, None} 
325              Specifies which metadata should be routed to `param` 
326   
327              - str: the name (or alias) of metadata given to a meta-estimator that 
328                should be routed to this parameter. 
329   
330              - True: requested 
331   
332              - False: not requested 
333   
334              - None: error if passed 
335          """
336          if not request_is_alias(alias) and not request_is_valid(alias):
337              raise ValueError(
338                  f"The alias you're setting for `{param}` should be either a "
339                  "valid identifier or one of {None, True, False}, but given "
340                  f"value is: `{alias}`"
341              )
342  
343          if alias == param:
344              alias = True
345  
346          if alias == UNUSED:
347              if param in self._requests:
348                  del self._requests[param]
349              else:
350                  raise ValueError(
351                      f"Trying to remove parameter {param} with UNUSED which doesn't"
352                      " exist."
353                  )
354          else:
355              self._requests[param] = alias
356  
357          return self
358  
359      def _get_param_names(self, return_alias):
360          """Get names of all metadata that can be consumed or routed by this method. 
361   
362          This method returns the names of all metadata, even the ``False`` 
363          ones. 
364   
365          Parameters 
366          ---------- 
367          return_alias : bool 
368              Controls whether original or aliased names should be returned. If 
369              ``False``, aliases are ignored and original names are returned. 
370   
371          Returns 
372          ------- 
373          names : set of str 
374              A set of strings with the names of all parameters. 
375          """
376          return set(
377              alias if return_alias and not request_is_valid(alias) else prop
378              for prop, alias in self._requests.items()
379              if not request_is_valid(alias) or alias is not False
380          )
381  
382      def _check_warnings(self, *, params):
383          """Check whether metadata is passed which is marked as WARN. 
384   
385          If any metadata is passed which is marked as WARN, a warning is raised. 
386   
387          Parameters 
388          ---------- 
389          params : dict 
390              The metadata passed to a method. 
391          """
392          params = {} if params is None else params
393          warn_params = {
394              prop
395              for prop, alias in self._requests.items()
396              if alias == WARN and prop in params
397          }
398          for param in warn_params:
399              warn(
400                  f"Support for {param} has recently been added to this class. "
401                  "To maintain backward compatibility, it is ignored now. "
402                  f"Using `set_{self.method}_request({param}={{True, False}})` "
403                  "on this method of the class, you can set the request value "
404                  "to False to silence this warning, or to True to consume and "
405                  "use the metadata."
406              )
407  
408      def _route_params(self, params, parent, caller):
409          """Prepare the given parameters to be passed to the method. 
410   
411          The output of this method can be used directly as the input to the 
412          corresponding method as extra props. 
413   
414          Parameters 
415          ---------- 
416          params : dict 
417              A dictionary of provided metadata. 
418   
419          parent : object 
420              Parent class object, that routes the metadata. 
421   
422          caller : str 
423              Method from the parent class object, where the metadata is routed from. 
424   
425          Returns 
426          ------- 
427          params : Bunch 
428              A :class:`~sklearn.utils.Bunch` of {prop: value} which can be given to the 
429              corresponding method. 
430          """
431          self._check_warnings(params=params)
432          unrequested = dict()
433          args = {arg: value for arg, value in params.items() if value is not None}
434          res = Bunch()
435          for prop, alias in self._requests.items():
436              if alias is False or alias == WARN:
437                  continue
438              elif alias is True and prop in args:
439                  res[prop] = args[prop]
440              elif alias is None and prop in args:
441                  unrequested[prop] = args[prop]
442              elif alias in args:
443                  res[prop] = args[alias]
444          if unrequested:
445              if self.method in COMPOSITE_METHODS:
446                  callee_methods = COMPOSITE_METHODS[self.method]
447              else:
448                  callee_methods = [self.method]
449              set_requests_on = "".join(
450                  [
451                      f".set_{method}_request({{metadata}}=True/False)"
452                      for method in callee_methods
453                  ]
454              )
455              message = (
456                  f"[{', '.join([key for key in unrequested])}] are passed but are not"
457                  " explicitly set as requested or not requested for"
458                  f" {self.owner}.{self.method}, which is used within"
459                  f" {parent}.{caller}. Call `{self.owner}"
460                  + set_requests_on
461                  + "` for each metadata you want to request/ignore."
462              )
463              raise UnsetMetadataPassedError(
464                  message=message,
465                  unrequested_params=unrequested,
466                  routed_params=res,
467              )
468          return res
469  
470      def _consumes(self, params):
471          """Check whether the given parameters are consumed by this method. 
472   
473          Parameters 
474          ---------- 
475          params : iterable of str 
476              An iterable of parameters to check. 
477   
478          Returns 
479          ------- 
480          consumed : set of str 
481              A set of parameters which are consumed by this method. 
482          """
483          params = set(params)
484          res = set()
485          for prop, alias in self._requests.items():
486              if alias is True and prop in params:
487                  res.add(prop)
488              elif isinstance(alias, str) and alias in params:
489                  res.add(alias)
490          return res
491  
492      def _serialize(self):
493          """Serialize the object. 
494   
495          Returns 
496          ------- 
497          obj : dict 
498              A serialized version of the instance in the form of a dictionary. 
499          """
500          return self._requests
501  
502      def __repr__(self):
503          return str(self._serialize())
504  
505      def __str__(self):
506          return str(repr(self))

509  class MetadataRequest:
510      """Contains the metadata request info of a consumer. 
511   
512      Instances of `MethodMetadataRequest` are used in this class for each 
513      available method under `metadatarequest.{method}`. 
514   
515      Consumer-only classes such as simple estimators return a serialized 
516      version of this class as the output of `get_metadata_routing()`. 
517   
518      .. versionadded:: 1.3 
519   
520      Parameters 
521      ---------- 
522      owner : str 
523          The name of the object to which these requests belong. 
524      """
525  
526      # this is here for us to use this attribute's value instead of doing
527      # `isinstance` in our checks, so that we avoid issues when people vendor
528      # this file instead of using it directly from scikit-learn.
529      _type = "metadata_request"
530  
531      def __init__(self, owner):
532          self.owner = owner
533          for method in SIMPLE_METHODS:
534              setattr(
535                  self,
536                  method,
537                  MethodMetadataRequest(owner=owner, method=method),
538              )
539  
540      def consumes(self, method, params):
541          """Check whether the given parameters are consumed by the given method. 
542   
543          .. versionadded:: 1.4 
544   
545          Parameters 
546          ---------- 
547          method : str 
548              The name of the method to check. 
549   
550          params : iterable of str 
551              An iterable of parameters to check. 
552   
553          Returns 
554          ------- 
555          consumed : set of str 
556              A set of parameters which are consumed by the given method. 
557          """
558          return getattr(self, method)._consumes(params=params)
559  
560      def __getattr__(self, name):
561          # Called when the default attribute access fails with an AttributeError
562          # (either __getattribute__() raises an AttributeError because name is
563          # not an instance attribute or an attribute in the class tree for self;
564          # or __get__() of a name property raises AttributeError). This method
565          # should either return the (computed) attribute value or raise an
566          # AttributeError exception.
567          # https://docs.python.org/3/reference/datamodel.html#object.__getattr__
568          if name not in COMPOSITE_METHODS:
569              raise AttributeError(
570                  f"'{self.__class__.__name__}' object has no attribute '{name}'"
571              )
572  
573          requests = {}
574          for method in COMPOSITE_METHODS[name]:
575              mmr = getattr(self, method)
576              existing = set(requests.keys())
577              upcoming = set(mmr.requests.keys())
578              common = existing & upcoming
579              conflicts = [key for key in common if requests[key] != mmr._requests[key]]
580              if conflicts:
581                  raise ValueError(
582                      f"Conflicting metadata requests for {', '.join(conflicts)} while"
583                      f" composing the requests for {name}. Metadata with the same name"
584                      f" for methods {', '.join(COMPOSITE_METHODS[name])} should have the"
585                      " same request value."
586                  )
587              requests.update(mmr._requests)
588          return MethodMetadataRequest(owner=self.owner, method=name, requests=requests)
589

597          Parameters 
598          ----------

602          return_alias : bool

609          Returns

684  # Metadata Request for Routers
685  # ============================
686  # This section includes all objects required for MetadataRouter which is used
687  # in routers, returned by their ``get_metadata_routing``.
688  
689  # `RouterMappingPair` is used to store a (mapping, router) tuple where `mapping` is a
690  # `MethodMapping` object and `router` is the output of `get_metadata_routing`.
691  # `MetadataRouter` stores a collection of `RouterMappingPair` objects in its
692  # `_route_mappings` attribute.
693  RouterMappingPair = namedtuple("RouterMappingPair", ["mapping", "router"])
694  
695  # `MethodPair` is used to store a single method routing. `MethodMapping` stores a list
696  # of `MethodPair` objects in its `_routes` attribute.
697  MethodPair = namedtuple("MethodPair", ["caller", "callee"])
698  
699  
700  class MethodMapping:
701      """Stores the mapping between caller and callee methods for a router. 
702   
703      This class is primarily used in a ``get_metadata_routing()`` of a router 
704      object when defining the mapping between the router's methods and a sub-object (a 
705      sub-estimator or a scorer). 
706   
707      Iterating through an instance of this class yields 
708      ``MethodPair(caller, callee)`` instances. 
709   
710      .. versionadded:: 1.3 
711      """
712  
713      def __init__(self):
714          self._routes = []
715  
716      def __iter__(self):
717          return iter(self._routes)
718  
719      def add(self, *, caller, callee):
720          """Add a method mapping. 
721   
722          Parameters 
723          ---------- 
724   
725          caller : str 
726              Parent estimator's method name in which the ``callee`` is called. 
727   
728          callee : str 
729              Child object's method name. This method is called in ``caller``. 
730   
731          Returns 
732          ------- 
733          self : MethodMapping 
734              Returns self. 
735          """
736          if caller not in METHODS:
737              raise ValueError(
738                  f"Given caller:{caller} is not a valid method. Valid methods are:"
739                  f" {METHODS}"
740              )
741          if callee not in METHODS:
742              raise ValueError(
743                  f"Given callee:{callee} is not a valid method. Valid methods are:"
744                  f" {METHODS}"
745              )
746          self._routes.append(MethodPair(caller=caller, callee=callee))
747          return self
748  
749      def _serialize(self):
750          """Serialize the object. 
751   
752          Returns 
753          ------- 
754          obj : list 
755              A serialized version of the instance in the form of a list. 
756          """
757          result = list()
758          for route in self._routes:
759              result.append({"caller": route.caller, "callee": route.callee})
760          return result

769  class MetadataRouter:
770      """Stores and handles metadata routing for a router object. 
771   
772      This class is used by router objects to store and handle metadata routing. 
773      Routing information is stored as a dictionary of the form ``{"object_name": 
774      RouteMappingPair(method_mapping, routing_info)}``, where ``method_mapping`` 
775      is an instance of :class:`~sklearn.utils.metadata_routing.MethodMapping` and 
776      ``routing_info`` is either a 
777      :class:`~sklearn.utils.metadata_routing.MetadataRequest` or a 
778      :class:`~sklearn.utils.metadata_routing.MetadataRouter` instance. 
779   
780      .. versionadded:: 1.3 
781   
782      Parameters 
783      ---------- 
784      owner : str 
785          The name of the object to which these requests belong. 
786      """
787  
788      # this is here for us to use this attribute's value instead of doing
789      # `isinstance`` in our checks, so that we avoid issues when people vendor
790      # this file instead of using it directly from scikit-learn.
791      _type = "metadata_router"
792  
793      def __init__(self, owner):
794          self._route_mappings = dict()
795          # `_self_request` is used if the router is also a consumer.
796          # _self_request, (added using `add_self_request()`) is treated
797          # differently from the other objects which are stored in
798          # _route_mappings.
799          self._self_request = None
800          self.owner = owner

802      def add_self_request(self, obj):
803          """Add `self` (as a consumer) to the routing. 
804   
805          This method is used if the router is also a consumer, and hence the 
806          router itself needs to be included in the routing. The passed object 
807          can be an estimator or a 
808          :class:`~sklearn.utils.metadata_routing.MetadataRequest`. 
809   
810          A router should add itself using this method instead of `add` since it 
811          should be treated differently than the other objects to which metadata 
812          is routed by the router. 
813   
814          Parameters 
815          ---------- 
816          obj : object 
817              This is typically the router instance, i.e. `self` in a 
818              ``get_metadata_routing()`` implementation. It can also be a 
819              ``MetadataRequest`` instance. 
820   
821          Returns 
822          ------- 
823          self : MetadataRouter 
824              Returns `self`. 
825          """
826          if getattr(obj, "_type", None) == "metadata_request":
827              self._self_request = deepcopy(obj)
828          elif hasattr(obj, "_get_metadata_request"):
829              self._self_request = deepcopy(obj._get_metadata_request())

831              raise ValueError(
832                  "Given `obj` is neither a `MetadataRequest` nor does it implement the"
833                  " required API. Inheriting from `BaseEstimator` implements the required"
834                  " API."

838      def add(self, *, method_mapping, **objs):
839          """Add named objects with their corresponding method mapping. 
840   
841          Parameters 
842          ---------- 
843          method_mapping : MethodMapping 
844              The mapping between the child and the parent's methods. 
845

848              :func:`~sklearn.utils.metadata_routing.get_routing_for_object` on them. 
849   
850          Returns

852          self : MetadataRouter

859                  mapping=method_mapping, router=get_routing_for_object(obj)

863      def consumes(self, method, params):
864          """Check whether the given parameters are consumed by the given method. 
865   
866          .. versionadded:: 1.4 
867   
868          Parameters 
869          ---------- 
870          method : str 
871              The name of the method to check. 
872   
873          params : iterable of str 
874              An iterable of parameters to check. 
875   
876          Returns 
877          ------- 
878          consumed : set of str 
879              A set of parameters which are consumed by the given method. 
880          """
881          res = set()
882          if self._self_request:
883              res = res | self._self_request.consumes(method=method, params=params)
884  
885          for _, route_mapping in self._route_mappings.items():
886              for caller, callee in route_mapping.mapping:
887                  if caller == method:
888                      res = res | route_mapping.router.consumes(
889                          method=callee, params=params
890                      )
891  
892          return res

894      def _get_param_names(self, *, method, return_alias, ignore_self_request):
895          """Get names of all metadata that can be consumed or routed by specified \ 
896              method. 
897   
898          This method returns the names of all metadata, even the ``False`` 
899          ones. 
900   
901          Parameters 
902          ---------- 
903          method : str 
904              The name of the method for which metadata names are requested. 
905   
906          return_alias : bool 
907              Controls whether original or aliased names should be returned, 
908              which only applies to the stored `self`. If no `self` routing 
909              object is stored, this parameter has no effect. 
910   
911          ignore_self_request : bool 
912              If `self._self_request` should be ignored. This is used in `_route_params`. 
913              If ``True``, ``return_alias`` has no effect. 
914   
915          Returns 
916          ------- 
917          names : set of str 
918              A set of strings with the names of all parameters. 
919          """
920          res = set()
921          if self._self_request and not ignore_self_request:
922              res = res.union(
923                  self._self_request._get_param_names(
924                      method=method, return_alias=return_alias
925                  )
926              )
927  
928          for name, route_mapping in self._route_mappings.items():
929              for caller, callee in route_mapping.mapping:
930                  if caller == method:
931                      res = res.union(
932                          route_mapping.router._get_param_names(
933                              method=callee, return_alias=True, ignore_self_request=False
934                          )
935                      )
936          return res

938      def _route_params(self, *, params, method, parent, caller):
939          """Prepare the given parameters to be passed to the method. 
940   
941          This is used when a router is used as a child object of another router. 
942          The parent router then passes all parameters understood by the child 
943          object to it and delegates their validation to the child. 
944   
945          The output of this method can be used directly as the input to the 
946          corresponding method as extra props. 
947   
948          Parameters 
949          ---------- 
950          params : dict 
951              A dictionary of provided metadata. 
952   
953          method : str 
954              The name of the method for which the parameters are requested and 
955              routed. 
956   
957          parent : object 
958              Parent class object, that routes the metadata. 
959   
960          caller : str 
961              Method from the parent class object, where the metadata is routed from. 
962   
963          Returns 
964          ------- 
965          params : Bunch 
966              A :class:`~sklearn.utils.Bunch` of {prop: value} which can be given to the 
967              corresponding method. 
968          """
969          res = Bunch()
970          if self._self_request:
971              res.update(
972                  self._self_request._route_params(
973                      params=params,
974                      method=method,
975                      parent=parent,
976                      caller=caller,
977                  )
978              )
979  
980          param_names = self._get_param_names(
981              method=method, return_alias=True, ignore_self_request=True
982          )
983          child_params = {
984              key: value for key, value in params.items() if key in param_names
985          }
986          for key in set(res.keys()).intersection(child_params.keys()):
987              # conflicts are okay if the passed objects are the same, but it's
988              # an issue if they're different objects.
989              if child_params[key] is not res[key]:
990                  raise ValueError(
991                      f"In {self.owner}, there is a conflict on {key} between what is"
992                      " requested for this estimator and what is requested by its"
993                      " children. You can resolve this conflict by using an alias for"
994                      " the child estimator(s) requested metadata."
995                  )
996  
997          res.update(child_params)
998          return res

1000     def route_params(self, *, caller, params):
1001         """Return the input parameters requested by child objects. 
1002  
1003         The output of this method is a :class:`~sklearn.utils.Bunch`, which includes the 
1004         metadata for all methods of each child object that is used in the router's 
1005         `caller` method. 
1006  
1007         If the router is also a consumer, it also checks for warnings of 
1008         `self`'s/consumer's requested metadata. 
1009  
1010         Parameters 
1011         ---------- 
1012         caller : str 
1013             The name of the method for which the parameters are requested and 
1014             routed. If called inside the :term:`fit` method of a router, it 
1015             would be `"fit"`. 
1016  
1017         params : dict 
1018             A dictionary of provided metadata. 
1019  
1020         Returns 
1021         ------- 
1022         params : Bunch 
1023             A :class:`~sklearn.utils.Bunch` of the form 
1024             ``{"object_name": {"method_name": {params: value}}}`` which can be 
1025             used to pass the required metadata to corresponding methods or 
1026             corresponding child objects. 
1027         """
1028         if self._self_request:
1029             self._self_request._check_warnings(params=params, method=caller)
1030 
1031         res = Bunch()
1032         for name, route_mapping in self._route_mappings.items():
1033             router, mapping = route_mapping.router, route_mapping.mapping
1034 
1035             res[name] = Bunch()
1036             for _caller, _callee in mapping:
1037                 if _caller == caller:
1038                     res[name][_callee] = router._route_params(
1039                         params=params,
1040                         method=_callee,
1041                         parent=self.owner,
1042                         caller=caller,
1043                     )
1044         return res

1046     def validate_metadata(self, *, method, params):
1047         """Validate given metadata for a method. 
1048  
1049         This raises a ``TypeError`` if some of the passed metadata are not 
1050         understood by child objects. 
1051  
1052         Parameters 
1053         ---------- 
1054         method : str 
1055             The name of the method for which the parameters are requested and 
1056             routed. If called inside the :term:`fit` method of a router, it 
1057             would be `"fit"`. 
1058  
1059         params : dict 
1060             A dictionary of provided metadata. 
1061         """
1062         param_names = self._get_param_names(

1211 class RequestMethod:
1212     """ 
1213     A descriptor for request methods. 
1214  
1215     .. versionadded:: 1.3 
1216  
1217     Parameters 
1218     ---------- 
1219     name : str 
1220         The name of the method for which the request function should be 
1221         created, e.g. ``"fit"`` would create a ``set_fit_request`` function. 
1222  
1223     keys : list of str 
1224         A list of strings which are accepted parameters by the created 
1225         function, e.g. ``["sample_weight"]`` if the corresponding method 
1226         accepts it as a metadata. 
1227  
1228     validate_keys : bool, default=True 
1229         Whether to check if the requested parameters fit the actual parameters 
1230         of the method. 
1231  
1232     Notes 
1233     ----- 
1234     This class is a descriptor [1]_ and uses PEP-362 to set the signature of 
1235     the returned function [2]_. 
1236  
1237     References 
1238     ---------- 
1239     .. [1] https://docs.python.org/3/howto/descriptor.html 
1240  
1241     .. [2] https://www.python.org/dev/peps/pep-0362/ 
1242     """
1243 
1244     def __init__(self, name, keys, validate_keys=True):
1245         self.name = name
1246         self.keys = keys
1247         self.validate_keys = validate_keys
1248 
1249     def __get__(self, instance, owner):
1250         # we would want to have a method which accepts only the expected args
1251         def func(*args, **kw):
1252             """Updates the request for provided parameters 
1253  
1254             This docstring is overwritten below. 
1255             See REQUESTER_DOC for expected functionality 
1256             """
1257             if not _routing_enabled():
1258                 raise RuntimeError(
1259                     "This method is only available when metadata routing is enabled."
1260                     " You can enable it using"
1261                     " sklearn.set_config(enable_metadata_routing=True)."
1262                 )
1263 
1264             if self.validate_keys and (set(kw) - set(self.keys)):
1265                 raise TypeError(
1266                     f"Unexpected args: {set(kw) - set(self.keys)} in {self.name}. "
1267                     f"Accepted arguments are: {set(self.keys)}"
1268                 )
1269 
1270             # This makes it possible to use the decorated method as an unbound method,
1271             # for instance when monkeypatching.
1272             # https://github.com/scikit-learn/scikit-learn/issues/28632
1273             if instance is None:
1274                 _instance = args[0]
1275                 args = args[1:]
1276             else:
1277                 _instance = instance
1278 
1279             # Replicating python's behavior when positional args are given other than
1280             # `self`, and `self` is only allowed if this method is unbound.
1281             if args:
1282                 raise TypeError(
1283                     f"set_{self.name}_request() takes 0 positional argument but"
1284                     f" {len(args)} were given"
1285                 )
1286 
1287             requests = _instance._get_metadata_request()
1288             method_metadata_request = getattr(requests, self.name)
1289 
1290             for prop, alias in kw.items():
1291                 if alias is not UNCHANGED:
1292                     method_metadata_request.add_request(param=prop, alias=alias)
1293             _instance._metadata_request = requests
1294 
1295             return _instance
1296 
1297         # Now we set the relevant attributes of the function so that it seems
1298         # like a normal method to the end user, with known expected arguments.
1299         func.__name__ = f"set_{self.name}_request"
1300         params = [
1301             inspect.Parameter(
1302                 name="self",
1303                 kind=inspect.Parameter.POSITIONAL_OR_KEYWORD,
1304                 annotation=owner,
1305             )
1306         ]
1307         params.extend(
1308             [
1309                 inspect.Parameter(
1310                     k,
1311                     inspect.Parameter.KEYWORD_ONLY,
1312                     default=UNCHANGED,
1313                     annotation=Optional[Union[bool, None, str]],
1314                 )
1315                 for k in self.keys
1316             ]
1317         )
1318         func.__signature__ = inspect.Signature(
1319             params,
1320             return_annotation=owner,
1321         )
1322         doc = REQUESTER_DOC.format(method=self.name)
1323         for metadata in self.keys:
1324             doc += REQUESTER_DOC_PARAM.format(metadata=metadata, method=self.name)
1325         doc += REQUESTER_DOC_RETURN
1326         func.__doc__ = doc
1327         return func

1378         except Exception:

1388             if not len(mmr.requests):

1397     @classmethod
1398     def _build_request_for_signature(cls, router, method):
1399         """Build the `MethodMetadataRequest` for a method using its signature. 
1400  
1401         This method takes all arguments from the method signature and uses 
1402         ``None`` as their default request value, except ``X``, ``y``, ``Y``, 
1403         ``Xt``, ``yt``, ``*args``, and ``**kwargs``. 
1404  
1405         Parameters 
1406         ---------- 
1407         router : MetadataRequest 
1408             The parent object for the created `MethodMetadataRequest`. 
1409         method : str 
1410             The name of the method. 
1411  
1412         Returns 
1413         ------- 
1414         method_request : MethodMetadataRequest 
1415             The prepared request using the method's signature. 
1416         """
1417         mmr = MethodMetadataRequest(owner=cls.__name__, method=method)
1418         # Here we use `isfunction` instead of `ismethod` because calling `getattr`
1419         # on a class instead of an instance returns an unbound function.
1420         if not hasattr(cls, method) or not inspect.isfunction(getattr(cls, method)):
1421             return mmr
1422         # ignore the first parameter of the method, which is usually "self"
1423         params = list(inspect.signature(getattr(cls, method)).parameters.items())[1:]
1424         for pname, param in params:
1425             if pname in {"X", "y", "Y", "Xt", "yt"}:
1426                 continue
1427             if param.kind in {param.VAR_POSITIONAL, param.VAR_KEYWORD}:
1428                 continue
1429             mmr.add_request(
1430                 param=pname,
1431                 alias=None,
1432             )
1433         return mmr

1446             setattr(
1447                 requests,
1448                 method,
1449                 cls._build_request_for_signature(router=requests, method=method),
1450             )
1451 
1452         # Then overwrite those defaults with the ones provided in
1453         # __metadata_request__* attributes. Defaults set in
1454         # __metadata_request__* attributes take precedence over signature
1455         # sniffing.
1456 
1457         # need to go through the MRO since this is a class attribute and
1458         # ``vars`` doesn't report the parent class attributes. We go through
1459         # the reverse of the MRO so that child classes have precedence over
1460         # their parents.
1461         substr = "__metadata_request__"
1462         for base_class in reversed(inspect.getmro(cls)):
1463             for attr, value in vars(base_class).items():
1464                 if substr not in attr:
1465                     continue
1466                 # we don't check for attr.startswith() since python prefixes attrs
1467                 # starting with __ with the `_ClassName`.
1468                 method = attr[attr.index(substr) + len(substr) :]
1469                 for prop, alias in value.items():
1470                     # Here we add request values specified via those class attributes
1471                     # to the `MetadataRequest` object. Adding a request which already
1472                     # exists will override the previous one. Since we go through the
1473                     # MRO in reverse order, the one specified by the lowest most classes
1474                     # in the inheritance tree are the ones which take effect.
1475                     getattr(requests, method).add_request(param=prop, alias=alias)
1476 
1477         return requests
1478

1480         """Get requested data properties. 
1481  
1482         Please check :ref:`User Guide <metadata_routing>` on how the routing 
1483         mechanism works. 
1484  
1485         Returns 
1486         ------- 
1487         request : MetadataRequest 
1488             A :class:`~sklearn.utils.metadata_routing.MetadataRequest` instance. 
1489         """
1490         if hasattr(self, "_metadata_request"):
1491             requests = get_routing_for_object(self._metadata_request)
1492         else:
1493             requests = self._get_default_requests()
1494 
1495         return requests

1512 # Process Routing in Routers
1513 # ==========================
1514 # This is almost always the only method used in routers to process and route
1515 # given metadata. This is to minimize the boilerplate required in routers.
1516 
1517 
1518 # Here the first two arguments are positional only which makes everything
1519 # passed as keyword argument a metadata. The first two args also have an `_`
1520 # prefix to reduce the chances of name collisions with the passed metadata, and
1521 # since they're positional only, users will never type those underscores.
1522 def process_routing(_obj, _method, /, **kwargs):
1523     """Validate and route input parameters. 
1524  
1525     This function is used inside a router's method, e.g. :term:`fit`, 
1526     to validate the metadata and handle the routing. 
1527  
1528     Assuming this signature of a router's fit method: 
1529     ``fit(self, X, y, sample_weight=None, **fit_params)``, 
1530     a call to this function would be: 
1531     ``process_routing(self, "fit", sample_weight=sample_weight, **fit_params)``. 
1532  
1533     Note that if routing is not enabled and ``kwargs`` is empty, then it 
1534     returns an empty routing where ``process_routing(...).ANYTHING.ANY_METHOD`` 
1535     is always an empty dictionary. 
1536  
1537     .. versionadded:: 1.3 
1538  
1539     Parameters 
1540     ---------- 
1541     _obj : object 
1542         An object implementing ``get_metadata_routing``. Typically a 
1543         meta-estimator. 
1544  
1545     _method : str 
1546         The name of the router's method in which this function is called. 
1547  
1548     **kwargs : dict 
1549         Metadata to be routed. 
1550  
1551     Returns 
1552     ------- 
1553     routed_params : Bunch 
1554         A :class:`~utils.Bunch` of the form ``{"object_name": {"method_name": 
1555         {params: value}}}`` which can be used to pass the required metadata to 
1556         A :class:`~sklearn.utils.Bunch` of the form ``{"object_name": {"method_name": 
1557         {params: value}}}`` which can be used to pass the required metadata to 
1558         corresponding methods or corresponding child objects. The object names 
1559         are those defined in `obj.get_metadata_routing()`. 
1560     """
1561     if not kwargs:
1562         # If routing is not enabled and kwargs are empty, then we don't have to
1563         # try doing any routing, we can simply return a structure which returns
1564         # an empty dict on routed_params.ANYTHING.ANY_METHOD.
1565         class EmptyRequest:
1566             def get(self, name, default=None):
1567                 return Bunch(**{method: dict() for method in METHODS})
1568 
1569             def __getitem__(self, name):
1570                 return Bunch(**{method: dict() for method in METHODS})
1571 
1572             def __getattr__(self, name):
1573                 return Bunch(**{method: dict() for method in METHODS})
1574 
1575         return EmptyRequest()
1576 
1577     if not (hasattr(_obj, "get_metadata_routing") or isinstance(_obj, MetadataRouter)):
1578         raise AttributeError(
1579             f"The given object ({repr(_obj.__class__.__name__)}) needs to either"
1580             " implement the routing method `get_metadata_routing` or be a"
1581             " `MetadataRouter` instance."
1582         )
1583     if _method not in METHODS:
1584         raise TypeError(
1585             f"Can only route and process input on these methods: {METHODS}, "
1586             f"while the passed method is: {_method}."
1587         )
1588 
1589     request_routing = get_routing_for_object(_obj)
1590     request_routing.validate_metadata(params=kwargs, method=_method)
1591     routed_params = request_routing.route_params(params=kwargs, caller=_method)
1592 
1593     return routed_params
