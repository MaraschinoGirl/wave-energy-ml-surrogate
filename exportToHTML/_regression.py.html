16   class KNeighborsRegressor(KNeighborsMixin, RegressorMixin, NeighborsBase):
17       """Regression based on k-nearest neighbors. 
18    
19       The target is predicted by local interpolation of the targets 
20       associated of the nearest neighbors in the training set. 
21    
22       Read more in the :ref:`User Guide <regression>`. 
23    
24       .. versionadded:: 0.9 
25    
26       Parameters 
27       ---------- 
28       n_neighbors : int, default=5 
29           Number of neighbors to use by default for :meth:`kneighbors` queries. 
30    
31       weights : {'uniform', 'distance'}, callable or None, default='uniform' 
32           Weight function used in prediction.  Possible values: 
33    
34           - 'uniform' : uniform weights.  All points in each neighborhood 
35             are weighted equally. 
36           - 'distance' : weight points by the inverse of their distance. 
37             in this case, closer neighbors of a query point will have a 
38             greater influence than neighbors which are further away. 
39           - [callable] : a user-defined function which accepts an 
40             array of distances, and returns an array of the same shape 
41             containing the weights. 
42    
43           Uniform weights are used by default. 
44    
45           See the following example for a demonstration of the impact of 
46           different weighting schemes on predictions: 
47           :ref:`sphx_glr_auto_examples_neighbors_plot_regression.py`. 
48    
49       algorithm : {'auto', 'ball_tree', 'kd_tree', 'brute'}, default='auto' 
50           Algorithm used to compute the nearest neighbors: 
51    
52           - 'ball_tree' will use :class:`BallTree` 
53           - 'kd_tree' will use :class:`KDTree` 
54           - 'brute' will use a brute-force search. 
55           - 'auto' will attempt to decide the most appropriate algorithm 
56             based on the values passed to :meth:`fit` method. 
57    
58           Note: fitting on sparse input will override the setting of 
59           this parameter, using brute force. 
60    
61       leaf_size : int, default=30 
62           Leaf size passed to BallTree or KDTree.  This can affect the 
63           speed of the construction and query, as well as the memory 
64           required to store the tree.  The optimal value depends on the 
65           nature of the problem. 
66    
67       p : float, default=2 
68           Power parameter for the Minkowski metric. When p = 1, this is 
69           equivalent to using manhattan_distance (l1), and euclidean_distance 
70           (l2) for p = 2. For arbitrary p, minkowski_distance (l_p) is used. 
71    
72       metric : str, DistanceMetric object or callable, default='minkowski' 
73           Metric to use for distance computation. Default is "minkowski", which 
74           results in the standard Euclidean distance when p = 2. See the 
75           documentation of `scipy.spatial.distance 
76           <https://docs.scipy.org/doc/scipy/reference/spatial.distance.html>`_ and 
77           the metrics listed in 
78           :class:`~sklearn.metrics.pairwise.distance_metrics` for valid metric 
79           values. 
80    
81           If metric is "precomputed", X is assumed to be a distance matrix and 
82           must be square during fit. X may be a :term:`sparse graph`, in which 
83           case only "nonzero" elements may be considered neighbors. 
84    
85           If metric is a callable function, it takes two arrays representing 1D 
86           vectors as inputs and must return one value indicating the distance 
87           between those vectors. This works for Scipy's metrics, but is less 
88           efficient than passing the metric name as a string. 
89    
90           If metric is a DistanceMetric object, it will be passed directly to 
91           the underlying computation routines. 
92    
93       metric_params : dict, default=None 
94           Additional keyword arguments for the metric function. 
95    
96       n_jobs : int, default=None 
97           The number of parallel jobs to run for neighbors search. 
98           ``None`` means 1 unless in a :obj:`joblib.parallel_backend` context. 
99           ``-1`` means using all processors. See :term:`Glossary <n_jobs>` 
100          for more details. 
101          Doesn't affect :meth:`fit` method. 
102   
103      Attributes 
104      ---------- 
105      effective_metric_ : str or callable 
106          The distance metric to use. It will be same as the `metric` parameter 
107          or a synonym of it, e.g. 'euclidean' if the `metric` parameter set to 
108          'minkowski' and `p` parameter set to 2. 
109   
110      effective_metric_params_ : dict 
111          Additional keyword arguments for the metric function. For most metrics 
112          will be same with `metric_params` parameter, but may also contain the 
113          `p` parameter value if the `effective_metric_` attribute is set to 
114          'minkowski'. 
115   
116      n_features_in_ : int 
117          Number of features seen during :term:`fit`. 
118   
119          .. versionadded:: 0.24 
120   
121      feature_names_in_ : ndarray of shape (`n_features_in_`,) 
122          Names of features seen during :term:`fit`. Defined only when `X` 
123          has feature names that are all strings. 
124   
125          .. versionadded:: 1.0 
126   
127      n_samples_fit_ : int 
128          Number of samples in the fitted data. 
129   
130      See Also 
131      -------- 
132      NearestNeighbors : Unsupervised learner for implementing neighbor searches. 
133      RadiusNeighborsRegressor : Regression based on neighbors within a fixed radius. 
134      KNeighborsClassifier : Classifier implementing the k-nearest neighbors vote. 
135      RadiusNeighborsClassifier : Classifier implementing 
136          a vote among neighbors within a given radius. 
137   
138      Notes 
139      ----- 
140      See :ref:`Nearest Neighbors <neighbors>` in the online documentation 
141      for a discussion of the choice of ``algorithm`` and ``leaf_size``. 
142   
143      .. warning:: 
144   
145         Regarding the Nearest Neighbors algorithms, if it is found that two 
146         neighbors, neighbor `k+1` and `k`, have identical distances but 
147         different labels, the results will depend on the ordering of the 
148         training data. 
149   
150      https://en.wikipedia.org/wiki/K-nearest_neighbors_algorithm 
151   
152      Examples 
153      -------- 
154      >>> X = [[0], [1], [2], [3]] 
155      >>> y = [0, 0, 1, 1] 
156      >>> from sklearn.neighbors import KNeighborsRegressor 
157      >>> neigh = KNeighborsRegressor(n_neighbors=2) 
158      >>> neigh.fit(X, y) 
159      KNeighborsRegressor(...) 
160      >>> print(neigh.predict([[1.5]])) 
161      [0.5] 
162      """
163  
164      _parameter_constraints: dict = {
165          **NeighborsBase._parameter_constraints,
166          "weights": [StrOptions({"uniform", "distance"}), callable, None],
167      }
168      _parameter_constraints["metric"].append(DistanceMetric)
169      _parameter_constraints.pop("radius")

171      def __init__(
172          self,
173          n_neighbors=5,
174          *,
175          weights="uniform",
176          algorithm="auto",
177          leaf_size=30,
178          p=2,
179          metric="minkowski",
180          metric_params=None,
181          n_jobs=None,
182      ):
183          super().__init__(
184              n_neighbors=n_neighbors,
185              algorithm=algorithm,
186              leaf_size=leaf_size,
187              metric=metric,
188              p=p,
189              metric_params=metric_params,
190              n_jobs=n_jobs,
191          )
192          self.weights = weights
193  
194      def __sklearn_tags__(self):
195          tags = super().__sklearn_tags__()
196          # For cross-validation routines to split data correctly
197          tags.input_tags.pairwise = self.metric == "precomputed"
198          return tags

200      @_fit_context(
201          # KNeighborsRegressor.metric is not validated yet
202          prefer_skip_nested_validation=False
203      )
204      def fit(self, X, y):
205          """Fit the k-nearest neighbors regressor from the training dataset. 
206   
207          Parameters 
208          ---------- 
209          X : {array-like, sparse matrix} of shape (n_samples, n_features) or \ 
210                  (n_samples, n_samples) if metric='precomputed' 
211              Training data. 
212   
213          y : {array-like, sparse matrix} of shape (n_samples,) or \ 
214                  (n_samples, n_outputs) 
215              Target values. 
216   
217          Returns 
218          ------- 
219          self : KNeighborsRegressor 
220              The fitted k-nearest neighbors regressor. 
221          """
222          return self._fit(X, y)
223  
224      def predict(self, X):
225          """Predict the target for the provided data. 
226   
227          Parameters 
228          ---------- 
229          X : {array-like, sparse matrix} of shape (n_queries, n_features), \ 
230                  or (n_queries, n_indexed) if metric == 'precomputed', or None 
231              Test samples. If `None`, predictions for all indexed points are 
232              returned; in this case, points are not considered their own 
233              neighbors. 
234   
235          Returns 
236          ------- 
237          y : ndarray of shape (n_queries,) or (n_queries, n_outputs), dtype=int 
238              Target values. 
239          """
240          if self.weights == "uniform":
241              # In that case, we do not need the distances to perform
242              # the weighting so we do not compute them.
243              neigh_ind = self.kneighbors(X, return_distance=False)
244              neigh_dist = None
245          else:
246              neigh_dist, neigh_ind = self.kneighbors(X)
247  
248          weights = _get_weights(neigh_dist, self.weights)
249  
250          _y = self._y
251          if _y.ndim == 1:
252              _y = _y.reshape((-1, 1))
253  
254          if weights is None:
255              y_pred = np.mean(_y[neigh_ind], axis=1)
256          else:
257              y_pred = np.empty((neigh_dist.shape[0], _y.shape[1]), dtype=np.float64)
258              denom = np.sum(weights, axis=1)
259  
260              for j in range(_y.shape[1]):
261                  num = np.sum(_y[neigh_ind, j] * weights, axis=1)
262                  y_pred[:, j] = num / denom
263  
264          if self._y.ndim == 1:
265              y_pred = y_pred.ravel()
266  
267          return y_pred

410          self,
411          radius=1.0,
412          *,
413          weights="uniform",
414          algorithm="auto",
415          leaf_size=30,
416          p=2,
417          metric="minkowski",
418          metric_params=None,

424              leaf_size=leaf_size,

426              metric=metric,
427              metric_params=metric_params,
428              n_jobs=n_jobs,

432      @_fit_context(
433          # RadiusNeighborsRegressor.metric is not validated yet
434          prefer_skip_nested_validation=False
435      )
436      def fit(self, X, y):
437          """Fit the radius neighbors regressor from the training dataset. 
438   
439          Parameters 
440          ---------- 
441          X : {array-like, sparse matrix} of shape (n_samples, n_features) or \ 
442                  (n_samples, n_samples) if metric='precomputed' 
443              Training data. 
444   
445          y : {array-like, sparse matrix} of shape (n_samples,) or \ 
446                  (n_samples, n_outputs) 
447              Target values. 
448   
449          Returns 
450          ------- 
451          self : RadiusNeighborsRegressor 
452              The fitted radius neighbors regressor. 
453          """
454          return self._fit(X, y)
455  
456      def predict(self, X):
457          """Predict the target for the provided data. 
458   
459          Parameters 
460          ---------- 
461          X : {array-like, sparse matrix} of shape (n_queries, n_features), \ 
462                  or (n_queries, n_indexed) if metric == 'precomputed', or None 
463              Test samples. If `None`, predictions for all indexed points are 
464              returned; in this case, points are not considered their own 
465              neighbors. 
466   
467          Returns 
468          ------- 
469          y : ndarray of shape (n_queries,) or (n_queries, n_outputs), \ 
470                  dtype=double 
471              Target values. 
472          """
473          neigh_dist, neigh_ind = self.radius_neighbors(X)
474  
475          weights = _get_weights(neigh_dist, self.weights)
476  
477          _y = self._y
478          if _y.ndim == 1:
479              _y = _y.reshape((-1, 1))
480  
481          empty_obs = np.full_like(_y[0], np.nan)
482  
483          if weights is None:
484              y_pred = np.array(
485                  [
486                      np.mean(_y[ind, :], axis=0) if len(ind) else empty_obs
487                      for (i, ind) in enumerate(neigh_ind)
488                  ]
489              )
490  
491          else:
492              y_pred = np.array(
493                  [
494                      (
495                          np.average(_y[ind, :], axis=0, weights=weights[i])
496                          if len(ind)
497                          else empty_obs
498                      )
499                      for (i, ind) in enumerate(neigh_ind)
500                  ]
501              )
502  
503          if np.any(np.isnan(y_pred)):
504              empty_warning_msg = (
505                  "One or more samples have no neighbors "
506                  "within specified radius; predicting NaN."
507              )
508              warnings.warn(empty_warning_msg)
509  
510          if self._y.ndim == 1:
511              y_pred = y_pred.ravel()
512  
513          return y_pred
