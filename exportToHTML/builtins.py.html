34   def I(x):
35       """The identity function. Simply returns its input unchanged. 
36    
37       Since Patsy's formula parser ignores anything inside a function call 
38       syntax, this is useful to 'hide' arithmetic operations from it. For 
39       instance:: 
40    
41         y ~ x1 + x2 
42    
43       has ``x1`` and ``x2`` as two separate predictors. But in:: 
44    
45         y ~ I(x1 + x2) 
46    
47       we instead have a single predictor, defined to be the sum of ``x1`` and 
48       ``x2``."""
49       return x

57   def Q(name):
58       """A way to 'quote' variable names, especially ones that do not otherwise 
59       meet Python's variable name rules. 
60    
61       If ``x`` is a variable, ``Q("x")`` returns the value of ``x``. (Note that 
62       ``Q`` takes the *string* ``"x"``, not the value of ``x`` itself.) This 
63       works even if instead of ``x``, we have a variable name that would not 
64       otherwise be legal in Python. 
65    
66       For example, if you have a column of data named ``weight.in.kg``, then you 
67       can't write:: 
68    
69         y ~ weight.in.kg 
70    
71       because Python will try to find a variable named ``weight``, that has an 
72       attribute named ``in``, that has an attribute named ``kg``. (And worse 
73       yet, ``in`` is a reserved word, which makes this example doubly broken.) 
74       Instead, write:: 
75    
76         y ~ Q("weight.in.kg") 
77    
78       and all will be well. Note, though, that this requires embedding a Python 
79       string inside your formula, which may require some care with your quote 
80       marks. Some standard options include:: 
81    
82         my_fit_function("y ~ Q('weight.in.kg')", ...) 
83         my_fit_function('y ~ Q("weight.in.kg")', ...) 
84         my_fit_function("y ~ Q(\\"weight.in.kg\\")", ...) 
85    
86       Note also that ``Q`` is an ordinary Python function, which means that you 
87       can use it in more complex expressions. For example, this is a legal 
88       formula:: 
89    
90         y ~ np.sqrt(Q("weight.in.kg")) 
91       """
92       from patsy.eval import EvalEnvironment
93   
94       env = EvalEnvironment.capture(1)
95       try:
96           return env.namespace[name]
97       except KeyError:
98           raise NameError("no data named %r found" % (name,))
