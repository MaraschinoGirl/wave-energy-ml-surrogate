2    Vendored library policy 
3    ======================= 
4    
5    To simplify Bleach development, we're now vendoring certain libraries that
6    we use.
7    
8    Vendored libraries must follow these rules:
9    
10   1. Vendored libraries must be pure Python--no compiling.
11   2. Source code for the libary is included in this directory.
12   3. License must be included in this repo and in the Bleach distribution.
13   4. Requirements of the library become requirements of Bleach.
14   5. No modifications to the library may be made.
15   
16   
17   Adding/Updating a vendored library 
18   ================================== 
19   
20   Way to vendor a library or update a version:
21   
22   1. Update ``vendor.txt`` with the library, version, and hash. You can use
23      `hashin <https://pypi.org/project/hashin/>`_.
24   2. Remove all old files and directories of the old version.
25   3. Run ``pip_install_vendor.sh`` and check everything it produced in including
26      the ``.dist-info`` directory and contents.
27   4. Update the bleach minor version in the next release.
28   
29   
30   Reviewing a change involving a vendored library 
31   =============================================== 
32   
33   Way to verify a vendored library addition/update:
34   
35   1. Pull down the branch.
36   2. Delete all the old files and directories of the old version.
37   3. Run ``pip_install_vendor.sh``.
38   4. Run ``git diff`` and verify there are no changes.
39   
40   
41   NB: the current ``vendor.txt`` was generated with pip 20.2.3, which might be necessary to reproduce the dist-info
42   
43   
44   Removing/Unvendoring a vendored library 
45   ======================================= 
46   
47   A vendored library might be removed for any of the following reasons:
48   
49   * it violates the vendoring policy (e.g. an incompatible license
50     change)
51   * a suitable replacement is found
52   * bleach has the resources to test and QA new bleach releases against
53     multiple versions of the previously vendored library
54   
55   To unvendor a library:
56   
57   1. Remove the library and its hashes from ``vendor.txt``.
58   2. Remove library files and directories from this directory.
59   3. Run ``install_vendor.sh`` and check the previously vendored library including
60      the ``.dist-info`` directory and contents is not installed.
61   4. Update the bleach minor version in the next release.
