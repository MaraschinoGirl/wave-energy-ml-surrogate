5    @keras_export("keras.layers.ConvLSTM1D")
6    class ConvLSTM1D(ConvLSTM):
7        """1D Convolutional LSTM. 
8     
9        Similar to an LSTM layer, but the input transformations 
10       and recurrent transformations are both convolutional. 
11    
12       Args: 
13           filters: int, the dimension of the output space (the number of filters 
14               in the convolution). 
15           kernel_size: int or tuple/list of 1 integer, specifying the size of 
16               the convolution window. 
17           strides: int or tuple/list of 1 integer, specifying the stride length 
18               of the convolution. `strides > 1` is incompatible with 
19               `dilation_rate > 1`. 
20           padding: string, `"valid"` or `"same"` (case-insensitive). 
21               `"valid"` means no padding. `"same"` results in padding evenly to 
22               the left/right or up/down of the input such that output has the 
23               same height/width dimension as the input. 
24           data_format: string, either `"channels_last"` or `"channels_first"`. 
25               The ordering of the dimensions in the inputs. `"channels_last"` 
26               corresponds to inputs with shape `(batch, steps, features)` 
27               while `"channels_first"` corresponds to inputs with shape 
28               `(batch, features, steps)`. It defaults to the `image_data_format` 
29               value found in your Keras config file at `~/.keras/keras.json`. 
30               If you never set it, then it will be `"channels_last"`. 
31           dilation_rate: int or tuple/list of 1 integers, specifying the dilation 
32               rate to use for dilated convolution. 
33           activation: Activation function to use. By default hyperbolic tangent 
34               activation function is applied (`tanh(x)`). 
35           recurrent_activation: Activation function to use for the recurrent step. 
36           use_bias: Boolean, whether the layer uses a bias vector. 
37           kernel_initializer: Initializer for the `kernel` weights matrix, 
38               used for the linear transformation of the inputs. 
39           recurrent_initializer: Initializer for the `recurrent_kernel` weights 
40               matrix, used for the linear transformation of the recurrent state. 
41           bias_initializer: Initializer for the bias vector. 
42           unit_forget_bias: Boolean. If `True`, add 1 to the bias of 
43               the forget gate at initialization. 
44               Use in combination with `bias_initializer="zeros"`. 
45               This is recommended in [Jozefowicz et al., 2015]( 
46               http://www.jmlr.org/proceedings/papers/v37/jozefowicz15.pdf) 
47           kernel_regularizer: Regularizer function applied to the `kernel` weights 
48               matrix. 
49           recurrent_regularizer: Regularizer function applied to the 
50               `recurrent_kernel` weights matrix. 
51           bias_regularizer: Regularizer function applied to the bias vector. 
52           activity_regularizer: Regularizer function applied to. 
53           kernel_constraint: Constraint function applied to the `kernel` weights 
54               matrix. 
55           recurrent_constraint: Constraint function applied to the 
56               `recurrent_kernel` weights matrix. 
57           bias_constraint: Constraint function applied to the bias vector. 
58           dropout: Float between 0 and 1. Fraction of the units to drop for the 
59               linear transformation of the inputs. 
60           recurrent_dropout: Float between 0 and 1. Fraction of the units to drop 
61               for the linear transformation of the recurrent state. 
62           seed: Random seed for dropout. 
63           return_sequences: Boolean. Whether to return the last output 
64               in the output sequence, or the full sequence. Default: `False`. 
65           return_state: Boolean. Whether to return the last state in addition 
66               to the output. Default: `False`. 
67           go_backwards: Boolean (default: `False`). 
68               If `True`, process the input sequence backwards and return the 
69               reversed sequence. 
70           stateful: Boolean (default False). If `True`, the last state 
71               for each sample at index i in a batch will be used as initial 
72               state for the sample of index i in the following batch. 
73           unroll: Boolean (default: `False`). 
74               If `True`, the network will be unrolled, 
75               else a symbolic loop will be used. 
76               Unrolling can speed-up a RNN, 
77               although it tends to be more memory-intensive. 
78               Unrolling is only suitable for short sequences. 
79    
80    
81       Call arguments: 
82           inputs: A 4D tensor. 
83           initial_state: List of initial state tensors to be passed to the first 
84               call of the cell. 
85           mask: Binary tensor of shape `(samples, timesteps)` indicating whether a 
86               given timestep should be masked. 
87           training: Python boolean indicating whether the layer should behave in 
88               training mode or in inference mode. 
89               This is only relevant if `dropout` or `recurrent_dropout` are set. 
90    
91       Input shape: 
92    
93       - If `data_format="channels_first"`: 
94           4D tensor with shape: `(samples, time, channels, rows)` 
95       - If `data_format="channels_last"`: 
96           4D tensor with shape: `(samples, time, rows, channels)` 
97    
98       Output shape: 
99    
100      - If `return_state`: a list of tensors. The first tensor is the output. 
101          The remaining tensors are the last states, 
102          each 3D tensor with shape: `(samples, filters, new_rows)` if 
103          `data_format='channels_first'` 
104          or shape: `(samples, new_rows, filters)` if 
105          `data_format='channels_last'`. 
106          `rows` values might have changed due to padding. 
107      - If `return_sequences`: 4D tensor with shape: `(samples, timesteps, 
108          filters, new_rows)` if data_format='channels_first' 
109          or shape: `(samples, timesteps, new_rows, filters)` if 
110          `data_format='channels_last'`. 
111      - Else, 3D tensor with shape: `(samples, filters, new_rows)` if 
112          `data_format='channels_first'` 
113          or shape: `(samples, new_rows, filters)` if 
114          `data_format='channels_last'`. 
115   
116      References: 
117   
118      - [Shi et al., 2015](http://arxiv.org/abs/1506.04214v1) 
119          (the current implementation does not include the feedback loop on the 
120          cells output). 
121      """
