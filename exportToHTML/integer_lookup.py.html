10   @keras_export("keras.layers.IntegerLookup")
11   class IntegerLookup(IndexLookup):
12       """A preprocessing layer that maps integers to (possibly encoded) indices. 
13    
14       This layer maps a set of arbitrary integer input tokens into indexed integer 
15       output via a table-based vocabulary lookup. The layer's output indices will 
16       be contiguously arranged up to the maximum vocab size, even if the input 
17       tokens are non-continguous or unbounded. The layer supports multiple options 
18       for encoding the output via `output_mode`, and has optional support for 
19       out-of-vocabulary (OOV) tokens and masking. 
20    
21       The vocabulary for the layer must be either supplied on construction or 
22       learned via `adapt()`. During `adapt()`, the layer will analyze a data set, 
23       determine the frequency of individual integer tokens, and create a 
24       vocabulary from them. If the vocabulary is capped in size, the most frequent 
25       tokens will be used to create the vocabulary and all others will be treated 
26       as OOV. 
27    
28       There are two possible output modes for the layer.  When `output_mode` is 
29       `"int"`, input integers are converted to their index in the vocabulary (an 
30       integer).  When `output_mode` is `"multi_hot"`, `"count"`, or `"tf_idf"`, 
31       input integers are encoded into an array where each dimension corresponds to 
32       an element in the vocabulary. 
33    
34       The vocabulary can optionally contain a mask token as well as an OOV token 
35       (which can optionally occupy multiple indices in the vocabulary, as set 
36       by `num_oov_indices`). 
37       The position of these tokens in the vocabulary is fixed. When `output_mode` 
38       is `"int"`, the vocabulary will begin with the mask token at index 0, 
39       followed by OOV indices, followed by the rest of the vocabulary. When 
40       `output_mode` is `"multi_hot"`, `"count"`, or `"tf_idf"` the vocabulary will 
41       begin with OOV indices and instances of the mask token will be dropped. 
42    
43       **Note:** This layer uses TensorFlow internally. It cannot 
44       be used as part of the compiled computation graph of a model with 
45       any backend other than TensorFlow. 
46       It can however be used with any backend when running eagerly. 
47       It can also always be used as part of an input preprocessing pipeline 
48       with any backend (outside the model itself), which is how we recommend 
49       to use this layer. 
50    
51       **Note:** This layer is safe to use inside a `tf.data` pipeline 
52       (independently of which backend you're using). 
53    
54       Args: 
55           max_tokens: Maximum size of the vocabulary for this layer. This should 
56               only be specified when adapting the vocabulary or when setting 
57               `pad_to_max_tokens=True`. If None, there is no cap on the size of 
58               the vocabulary. Note that this size includes the OOV 
59               and mask tokens. Defaults to `None`. 
60           num_oov_indices: The number of out-of-vocabulary tokens to use. 
61               If this value is more than 1, OOV inputs are modulated to 
62               determine their OOV value. 
63               If this value is 0, OOV inputs will cause an error when calling 
64               the layer. Defaults to `1`. 
65           mask_token: An integer token that represents masked inputs. When 
66               `output_mode` is `"int"`, the token is included in vocabulary 
67               and mapped to index 0. In other output modes, 
68               the token will not appear in the vocabulary and instances 
69               of the mask token in the input will be dropped. 
70               If set to None, no mask term will be added. Defaults to `None`. 
71           oov_token: Only used when `invert` is `True`. The token to return 
72               for OOV indices. Defaults to `-1`. 
73           vocabulary: Optional. Either an array of integers or a string path to a 
74               text file. If passing an array, can pass a tuple, list, 
75               1D NumPy array, or 1D tensor containing the integer vocbulary terms. 
76               If passing a file path, the file should contain one line per term 
77               in the vocabulary. If this argument is set, 
78               there is no need to `adapt()` the layer. 
79           vocabulary_dtype: The dtype of the vocabulary terms, for example 
80               `"int64"` or `"int32"`. Defaults to `"int64"`. 
81           idf_weights: Only valid when `output_mode` is `"tf_idf"`. 
82               A tuple, list, 1D NumPy array, or 1D tensor or the same length 
83               as the vocabulary, containing the floating point inverse document 
84               frequency weights, which will be multiplied by per sample term 
85               counts for the final TF-IDF weight. 
86               If the `vocabulary` argument is set, and `output_mode` is 
87               `"tf_idf"`, this argument must be supplied. 
88           invert: Only valid when `output_mode` is `"int"`. 
89               If `True`, this layer will map indices to vocabulary items 
90               instead of mapping vocabulary items to indices. 
91               Defaults to `False`. 
92           output_mode: Specification for the output of the layer. Values can be 
93               `"int"`, `"one_hot"`, `"multi_hot"`, `"count"`, or `"tf_idf"` 
94               configuring the layer as follows: 
95               - `"int"`: Return the vocabulary indices of the input tokens. 
96               - `"one_hot"`: Encodes each individual element in the input into an 
97                   array the same size as the vocabulary, 
98                   containing a 1 at the element index. If the last dimension 
99                   is size 1, will encode on that dimension. 
100                  If the last dimension is not size 1, will append a new 
101                  dimension for the encoded output. 
102              - `"multi_hot"`: Encodes each sample in the input into a single 
103                  array the same size as the vocabulary, 
104                  containing a 1 for each vocabulary term present in the sample. 
105                  Treats the last dimension as the sample dimension, 
106                  if input shape is `(..., sample_length)`, 
107                  output shape will be `(..., num_tokens)`. 
108              - `"count"`: As `"multi_hot"`, but the int array contains 
109                  a count of the number of times the token at that index 
110                  appeared in the sample. 
111              - `"tf_idf"`: As `"multi_hot"`, but the TF-IDF algorithm is 
112                  applied to find the value in each token slot. 
113              For `"int"` output, any shape of input and output is supported. 
114              For all other output modes, currently only output up to rank 2 
115              is supported. Defaults to `"int"`. 
116          pad_to_max_tokens: Only applicable when `output_mode` is `"multi_hot"`, 
117              `"count"`, or `"tf_idf"`. If `True`, the output will have 
118              its feature axis padded to `max_tokens` even if the number 
119              of unique tokens in the vocabulary is less than `max_tokens`, 
120              resulting in a tensor of shape `(batch_size, max_tokens)` 
121              regardless of vocabulary size. Defaults to `False`. 
122          sparse: Boolean. Only applicable to `"multi_hot"`, `"count"`, and 
123              `"tf_idf"` output modes. Only supported with TensorFlow 
124              backend. If `True`, returns a `SparseTensor` 
125              instead of a dense `Tensor`. Defaults to `False`. 
126   
127      Examples: 
128   
129      **Creating a lookup layer with a known vocabulary** 
130   
131      This example creates a lookup layer with a pre-existing vocabulary. 
132   
133      >>> vocab = [12, 36, 1138, 42] 
134      >>> data = np.array([[12, 1138, 42], [42, 1000, 36]])  # Note OOV tokens 
135      >>> layer = IntegerLookup(vocabulary=vocab) 
136      >>> layer(data) 
137      array([[1, 3, 4], 
138             [4, 0, 2]]) 
139   
140      **Creating a lookup layer with an adapted vocabulary** 
141   
142      This example creates a lookup layer and generates the vocabulary by 
143      analyzing the dataset. 
144   
145      >>> data = np.array([[12, 1138, 42], [42, 1000, 36]]) 
146      >>> layer = IntegerLookup() 
147      >>> layer.adapt(data) 
148      >>> layer.get_vocabulary() 
149      [-1, 42, 1138, 1000, 36, 12] 
150   
151      Note that the OOV token -1 have been added to the vocabulary. The remaining 
152      tokens are sorted by frequency (42, which has 2 occurrences, is first) then 
153      by inverse sort order. 
154   
155      >>> data = np.array([[12, 1138, 42], [42, 1000, 36]]) 
156      >>> layer = IntegerLookup() 
157      >>> layer.adapt(data) 
158      >>> layer(data) 
159      array([[5, 2, 1], 
160             [1, 3, 4]]) 
161   
162      **Lookups with multiple OOV indices** 
163   
164      This example demonstrates how to use a lookup layer with multiple OOV 
165      indices.  When a layer is created with more than one OOV index, any OOV 
166      tokens are hashed into the number of OOV buckets, distributing OOV tokens in 
167      a deterministic fashion across the set. 
168   
169      >>> vocab = [12, 36, 1138, 42] 
170      >>> data = np.array([[12, 1138, 42], [37, 1000, 36]]) 
171      >>> layer = IntegerLookup(vocabulary=vocab, num_oov_indices=2) 
172      >>> layer(data) 
173      array([[2, 4, 5], 
174             [1, 0, 3]]) 
175   
176      Note that the output for OOV token 37 is 1, while the output for OOV token 
177      1000 is 0. The in-vocab terms have their output index increased by 1 from 
178      earlier examples (12 maps to 2, etc) in order to make space for the extra 
179      OOV token. 
180   
181      **One-hot output** 
182   
183      Configure the layer with `output_mode='one_hot'`. Note that the first 
184      `num_oov_indices` dimensions in the ont_hot encoding represent OOV values. 
185   
186      >>> vocab = [12, 36, 1138, 42] 
187      >>> data = np.array([12, 36, 1138, 42, 7])  # Note OOV tokens 
188      >>> layer = IntegerLookup(vocabulary=vocab, output_mode='one_hot') 
189      >>> layer(data) 
190      array([[0., 1., 0., 0., 0.], 
191              [0., 0., 1., 0., 0.], 
192              [0., 0., 0., 1., 0.], 
193              [0., 0., 0., 0., 1.], 
194              [1., 0., 0., 0., 0.]], dtype=float32) 
195   
196      **Multi-hot output** 
197   
198      Configure the layer with `output_mode='multi_hot'`. Note that the first 
199      `num_oov_indices` dimensions in the multi_hot encoding represent OOV tokens 
200   
201      >>> vocab = [12, 36, 1138, 42] 
202      >>> data = np.array([[12, 1138, 42, 42], 
203      ...                  [42,    7, 36,  7]])  # Note OOV tokens 
204      >>> layer = IntegerLookup(vocabulary=vocab, output_mode='multi_hot') 
205      >>> layer(data) 
206      array([[0., 1., 0., 1., 1.], 
207             [1., 0., 1., 0., 1.]], dtype=float32) 
208   
209      **Token count output** 
210   
211      Configure the layer with `output_mode='count'`. As with multi_hot output, 
212      the first `num_oov_indices` dimensions in the output represent OOV tokens. 
213   
214      >>> vocab = [12, 36, 1138, 42] 
215      >>> data = np.array([[12, 1138, 42, 42], 
216      ...                  [42,    7, 36,  7]])  # Note OOV tokens 
217      >>> layer = IntegerLookup(vocabulary=vocab, output_mode='count') 
218      >>> layer(data) 
219      array([[0., 1., 0., 1., 2.], 
220             [2., 0., 1., 0., 1.]], dtype=float32) 
221   
222      **TF-IDF output** 
223   
224      Configure the layer with `output_mode='tf_idf'`. As with multi_hot output, 
225      the first `num_oov_indices` dimensions in the output represent OOV tokens. 
226   
227      Each token bin will output `token_count * idf_weight`, where the idf weights 
228      are the inverse document frequency weights per token. These should be 
229      provided along with the vocabulary. Note that the `idf_weight` for OOV 
230      tokens will default to the average of all idf weights passed in. 
231   
232      >>> vocab = [12, 36, 1138, 42] 
233      >>> idf_weights = [0.25, 0.75, 0.6, 0.4] 
234      >>> data = np.array([[12, 1138, 42, 42], 
235      ...                  [42,    7, 36,  7]])  # Note OOV tokens 
236      >>> layer = IntegerLookup( 
237      ...     output_mode='tf_idf', vocabulary=vocab, idf_weights=idf_weights) 
238      >>> layer(data) 
239      array([[0.  , 0.25, 0.  , 0.6 , 0.8 ], 
240              [1.0 , 0.  , 0.75, 0.  , 0.4 ]], dtype=float32) 
241   
242      To specify the idf weights for oov tokens, you will need to pass the entire 
243      vocabulary including the leading oov token. 
244   
245      >>> vocab = [-1, 12, 36, 1138, 42] 
246      >>> idf_weights = [0.9, 0.25, 0.75, 0.6, 0.4] 
247      >>> data = np.array([[12, 1138, 42, 42], 
248      ...                  [42,    7, 36,  7]])  # Note OOV tokens 
249      >>> layer = IntegerLookup( 
250      ...     output_mode='tf_idf', vocabulary=vocab, idf_weights=idf_weights) 
251      >>> layer(data) 
252      array([[0.  , 0.25, 0.  , 0.6 , 0.8 ], 
253              [1.8 , 0.  , 0.75, 0.  , 0.4 ]], dtype=float32) 
254   
255      When adapting the layer in `"tf_idf"` mode, each input sample will 
256      be considered a document, and IDF weight per token will be 
257      calculated as: 
258      `log(1 + num_documents / (1 + token_document_count))`. 
259   
260      **Inverse lookup** 
261   
262      This example demonstrates how to map indices to tokens using this layer. 
263      (You can also use `adapt()` with `inverse=True`, but for simplicity we'll 
264      pass the vocab in this example.) 
265   
266      >>> vocab = [12, 36, 1138, 42] 
267      >>> data = np.array([[1, 3, 4], [4, 0, 2]]) 
268      >>> layer = IntegerLookup(vocabulary=vocab, invert=True) 
269      >>> layer(data) 
270      array([[  12, 1138,   42], 
271             [  42,   -1,   36]]) 
272   
273      Note that the first index correspond to the oov token by default. 
274   
275      **Forward and inverse lookup pairs** 
276   
277      This example demonstrates how to use the vocabulary of a standard lookup 
278      layer to create an inverse lookup layer. 
279   
280      >>> vocab = [12, 36, 1138, 42] 
281      >>> data = np.array([[12, 1138, 42], [42, 1000, 36]]) 
282      >>> layer = IntegerLookup(vocabulary=vocab) 
283      >>> i_layer = IntegerLookup( 
284      ...     vocabulary=layer.get_vocabulary(), invert=True) 
285      >>> int_data = layer(data) 
286      >>> i_layer(int_data) 
287      array([[  12, 1138,   42], 
288             [  42,   -1,   36]]) 
289   
290      In this example, the input token 1000 resulted in an output of -1, since 
291      1000 was not in the vocabulary - it got represented as an OOV, and all OOV 
292      tokens are returned as -1 in the inverse layer. Also, note that for the 
293      inverse to work, you must have already set the forward layer vocabulary 
294      either directly or via `adapt()` before calling `get_vocabulary()`. 
295      """

297      def __init__(
298          self,
299          max_tokens=None,
300          num_oov_indices=1,
301          mask_token=None,
302          oov_token=-1,
303          vocabulary=None,
304          vocabulary_dtype="int64",
305          idf_weights=None,
306          invert=False,
307          output_mode="int",
308          sparse=False,
309          pad_to_max_tokens=False,
310          name=None,
311          **kwargs,
312      ):
313          if not tf.available:
314              raise ImportError(
315                  "Layer IntegerLookup requires TensorFlow. "
316                  "Install it via `pip install tensorflow`."
317              )
318          if max_tokens is not None and max_tokens <= 1:
319              raise ValueError(
320                  "If `max_tokens` is set for `IntegerLookup`, it must be "
321                  f"greater than 1. Received: max_tokens={max_tokens}"
322              )
323          if num_oov_indices < 0:
324              raise ValueError(
325                  "The value of `num_oov_indices` argument for `IntegerLookup` "
326                  "must >= 0. Received: num_oov_indices="
327                  f"{num_oov_indices}"
328              )
329          if sparse and backend.backend() != "tensorflow":
330              raise ValueError(
331                  "`sparse=True` can only be used with the TensorFlow backend."
332              )
333          if vocabulary_dtype != "int64":
334              raise ValueError(
335                  "Only `vocabulary_dtype='int64'` is supported "
336                  "at this time. Received: "
337                  f"vocabulary_dtype={vocabulary_dtype}"
338              )
339          super().__init__(
340              max_tokens=max_tokens,
341              num_oov_indices=num_oov_indices,
342              mask_token=mask_token,
343              oov_token=oov_token,
344              vocabulary=vocabulary,
345              vocabulary_dtype=vocabulary_dtype,
346              idf_weights=idf_weights,
347              invert=invert,
348              output_mode=output_mode,
349              sparse=sparse,
350              pad_to_max_tokens=pad_to_max_tokens,
351              name=name,
352              **kwargs,
353          )
354          self._convert_input_args = False
355          self._allow_non_tensor_positional_args = True
356          self.supports_jit = False

358      def adapt(self, data, steps=None):
359          """Computes a vocabulary of integer terms from tokens in a dataset. 
360   
361          Calling `adapt()` on an `IntegerLookup` layer is an alternative to 
362          passing in a precomputed vocabulary  on construction via the 
363          `vocabulary` argument.  An `IntegerLookup` layer should always be either 
364          adapted over a dataset or supplied with a vocabulary. 
365   
366          During `adapt()`, the layer will build a vocabulary of all integer 
367          tokens seen in the dataset, sorted by occurrence count, with ties broken 
368          by sort order of the tokens (high to low). At the end of `adapt()`, if 
369          `max_tokens` is set, the vocabulary will be truncated to `max_tokens` 
370          size. For example, adapting a layer with `max_tokens=1000` will compute 
371          the 1000 most frequent tokens occurring in the input dataset. If 
372          `output_mode='tf-idf'`, `adapt()` will also learn the document 
373          frequencies of each token in the input dataset. 
374   
375          Arguments: 
376              data: The data to train on. It can be passed either as a 
377                  batched `tf.data.Dataset`, as a list of integers, 
378                  or as a NumPy array. 
379              steps: Integer or `None`. 
380                  Total number of steps (batches of samples) to process. 
381                  If `data` is a `tf.data.Dataset`, and `steps` is `None`, 
382                  `adapt()` will run until the input dataset is exhausted. 
383                  When passing an infinitely 
384                  repeating dataset, you must specify the `steps` argument. This 
385                  argument is not supported with array inputs or list inputs. 
386          """
387          super().adapt(data, steps=steps)
388  
389      def get_config(self):
390          config = super().get_config()
391          if config["oov_token"] is not None:
392              config["oov_token"] = int(config["oov_token"])
393          if config["mask_token"] is not None:
394              config["mask_token"] = int(config["mask_token"])
395          if config["vocabulary"] is not None:
396              config["vocabulary"] = [int(v) for v in config["vocabulary"]]
397          return config
398  
399      def call(self, inputs):
400          if not isinstance(
401              inputs, (tf.Tensor, tf.RaggedTensor, np.ndarray, list, tuple)
402          ):
403              inputs = tf.convert_to_tensor(backend.convert_to_numpy(inputs))
404          outputs = super().call(inputs)
405          return backend_utils.convert_tf_tensor(outputs)
