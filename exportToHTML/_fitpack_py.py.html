13   def splprep(x, w=None, u=None, ub=None, ue=None, k=3, task=0, s=None, t=None,
14               full_output=0, nest=None, per=0, quiet=1):
15       """ 
16       Find the B-spline representation of an N-D curve. 
17    
18       .. legacy:: function 
19    
20           Specifically, we recommend using `make_splprep` in new code. 
21    
22       Given a list of N rank-1 arrays, `x`, which represent a curve in 
23       N-dimensional space parametrized by `u`, find a smooth approximating 
24       spline curve g(`u`). Uses the FORTRAN routine parcur from FITPACK. 
25    
26       Parameters 
27       ---------- 
28       x : array_like 
29           A list of sample vector arrays representing the curve. 
30       w : array_like, optional 
31           Strictly positive rank-1 array of weights the same length as `x[0]`. 
32           The weights are used in computing the weighted least-squares spline 
33           fit. If the errors in the `x` values have standard-deviation given by 
34           the vector d, then `w` should be 1/d. Default is ``ones(len(x[0]))``. 
35       u : array_like, optional 
36           An array of parameter values. If not given, these values are 
37           calculated automatically as ``M = len(x[0])``, where 
38    
39               v[0] = 0 
40    
41               v[i] = v[i-1] + distance(`x[i]`, `x[i-1]`) 
42    
43               u[i] = v[i] / v[M-1] 
44    
45       ub, ue : int, optional 
46           The end-points of the parameters interval.  Defaults to 
47           u[0] and u[-1]. 
48       k : int, optional 
49           Degree of the spline. Cubic splines are recommended. 
50           Even values of `k` should be avoided especially with a small s-value. 
51           ``1 <= k <= 5``, default is 3. 
52       task : int, optional 
53           If task==0 (default), find t and c for a given smoothing factor, s. 
54           If task==1, find t and c for another value of the smoothing factor, s. 
55           There must have been a previous call with task=0 or task=1 
56           for the same set of data. 
57           If task=-1 find the weighted least square spline for a given set of 
58           knots, t. 
59       s : float, optional 
60           A smoothing condition.  The amount of smoothness is determined by 
61           satisfying the conditions: ``sum((w * (y - g))**2,axis=0) <= s``, 
62           where g(x) is the smoothed interpolation of (x,y).  The user can 
63           use `s` to control the trade-off between closeness and smoothness 
64           of fit.  Larger `s` means more smoothing while smaller values of `s` 
65           indicate less smoothing. Recommended values of `s` depend on the 
66           weights, w.  If the weights represent the inverse of the 
67           standard-deviation of y, then a good `s` value should be found in 
68           the range ``(m-sqrt(2*m),m+sqrt(2*m))``, where m is the number of 
69           data points in x, y, and w. 
70       t : array, optional 
71           The knots needed for ``task=-1``. 
72           There must be at least ``2*k+2`` knots. 
73       full_output : int, optional 
74           If non-zero, then return optional outputs. 
75       nest : int, optional 
76           An over-estimate of the total number of knots of the spline to 
77           help in determining the storage space.  By default nest=m/2. 
78           Always large enough is nest=m+k+1. 
79       per : int, optional 
80          If non-zero, data points are considered periodic with period 
81          ``x[m-1] - x[0]`` and a smooth periodic spline approximation is 
82          returned.  Values of ``y[m-1]`` and ``w[m-1]`` are not used. 
83       quiet : int, optional 
84            Non-zero to suppress messages. 
85    
86       Returns 
87       ------- 
88       tck : tuple 
89           A tuple, ``(t,c,k)`` containing the vector of knots, the B-spline 
90           coefficients, and the degree of the spline. 
91       u : array 
92           An array of the values of the parameter. 
93       fp : float 
94           The weighted sum of squared residuals of the spline approximation. 
95       ier : int 
96           An integer flag about splrep success.  Success is indicated 
97           if ier<=0. If ier in [1,2,3] an error occurred but was not raised. 
98           Otherwise an error is raised. 
99       msg : str 
100          A message corresponding to the integer flag, ier. 
101   
102      See Also 
103      -------- 
104      splrep, splev, sproot, spalde, splint, 
105      bisplrep, bisplev 
106      UnivariateSpline, BivariateSpline 
107      BSpline 
108      make_interp_spline 
109   
110      Notes 
111      ----- 
112      See `splev` for evaluation of the spline and its derivatives. 
113      The number of dimensions N must be smaller than 11. 
114   
115      The number of coefficients in the `c` array is ``k+1`` less than the number 
116      of knots, ``len(t)``. This is in contrast with `splrep`, which zero-pads 
117      the array of coefficients to have the same length as the array of knots. 
118      These additional coefficients are ignored by evaluation routines, `splev` 
119      and `BSpline`. 
120   
121      References 
122      ---------- 
123      .. [1] P. Dierckx, "Algorithms for smoothing data with periodic and 
124          parametric splines, Computer Graphics and Image Processing", 
125          20 (1982) 171-184. 
126      .. [2] P. Dierckx, "Algorithms for smoothing data with periodic and 
127          parametric splines", report tw55, Dept. Computer Science, 
128          K.U.Leuven, 1981. 
129      .. [3] P. Dierckx, "Curve and surface fitting with splines", Monographs on 
130          Numerical Analysis, Oxford University Press, 1993. 
131   
132      Examples 
133      -------- 
134      Generate a discretization of a limacon curve in the polar coordinates: 
135   
136      >>> import numpy as np 
137      >>> phi = np.linspace(0, 2.*np.pi, 40) 
138      >>> r = 0.5 + np.cos(phi)         # polar coords 
139      >>> x, y = r * np.cos(phi), r * np.sin(phi)    # convert to cartesian 
140   
141      And interpolate: 
142   
143      >>> from scipy.interpolate import splprep, splev 
144      >>> tck, u = splprep([x, y], s=0) 
145      >>> new_points = splev(u, tck) 
146   
147      Notice that (i) we force interpolation by using ``s=0``, 
148      (ii) the parameterization, ``u``, is generated automatically. 
149      Now plot the result: 
150   
151      >>> import matplotlib.pyplot as plt 
152      >>> fig, ax = plt.subplots() 
153      >>> ax.plot(x, y, 'ro') 
154      >>> ax.plot(new_points[0], new_points[1], 'r-') 
155      >>> plt.show() 
156   
157      """
158  
159      res = _impl.splprep(x, w, u, ub, ue, k, task, s, t, full_output, nest, per,
160                          quiet)
161      return res

164  def splrep(x, y, w=None, xb=None, xe=None, k=3, task=0, s=None, t=None,
165             full_output=0, per=0, quiet=1):
166      """ 
167      Find the B-spline representation of a 1-D curve. 
168   
169      .. legacy:: function 
170   
171          Specifically, we recommend using `make_splrep` in new code. 
172   
173   
174      Given the set of data points ``(x[i], y[i])`` determine a smooth spline 
175      approximation of degree k on the interval ``xb <= x <= xe``. 
176   
177      Parameters 
178      ---------- 
179      x, y : array_like 
180          The data points defining a curve ``y = f(x)``. 
181      w : array_like, optional 
182          Strictly positive rank-1 array of weights the same length as `x` and `y`. 
183          The weights are used in computing the weighted least-squares spline 
184          fit. If the errors in the `y` values have standard-deviation given by the 
185          vector ``d``, then `w` should be ``1/d``. Default is ``ones(len(x))``. 
186      xb, xe : float, optional 
187          The interval to fit.  If None, these default to ``x[0]`` and ``x[-1]`` 
188          respectively. 
189      k : int, optional 
190          The degree of the spline fit. It is recommended to use cubic splines. 
191          Even values of `k` should be avoided especially with small `s` values. 
192          ``1 <= k <= 5``. 
193      task : {1, 0, -1}, optional 
194          If ``task==0``, find ``t`` and ``c`` for a given smoothing factor, `s`. 
195   
196          If ``task==1`` find ``t`` and ``c`` for another value of the smoothing factor, 
197          `s`. There must have been a previous call with ``task=0`` or ``task=1`` for 
198          the same set of data (``t`` will be stored an used internally) 
199   
200          If ``task=-1`` find the weighted least square spline for a given set of 
201          knots, ``t``. These should be interior knots as knots on the ends will be 
202          added automatically. 
203      s : float, optional 
204          A smoothing condition. The amount of smoothness is determined by 
205          satisfying the conditions: ``sum((w * (y - g))**2,axis=0) <= s`` where ``g(x)`` 
206          is the smoothed interpolation of ``(x,y)``. The user can use `s` to control 
207          the tradeoff between closeness and smoothness of fit. Larger `s` means 
208          more smoothing while smaller values of `s` indicate less smoothing. 
209          Recommended values of `s` depend on the weights, `w`. If the weights 
210          represent the inverse of the standard-deviation of `y`, then a good `s` 
211          value should be found in the range ``(m-sqrt(2*m),m+sqrt(2*m))`` where ``m`` is 
212          the number of datapoints in `x`, `y`, and `w`. default : ``s=m-sqrt(2*m)`` if 
213          weights are supplied. ``s = 0.0`` (interpolating) if no weights are 
214          supplied. 
215      t : array_like, optional 
216          The knots needed for ``task=-1``. If given then task is automatically set 
217          to ``-1``. 
218      full_output : bool, optional 
219          If non-zero, then return optional outputs. 
220      per : bool, optional 
221          If non-zero, data points are considered periodic with period ``x[m-1]`` - 
222          ``x[0]`` and a smooth periodic spline approximation is returned. Values of 
223          ``y[m-1]`` and ``w[m-1]`` are not used. 
224          The default is zero, corresponding to boundary condition 'not-a-knot'. 
225      quiet : bool, optional 
226          Non-zero to suppress messages. 
227   
228      Returns 
229      ------- 
230      tck : tuple 
231          A tuple ``(t,c,k)`` containing the vector of knots, the B-spline 
232          coefficients, and the degree of the spline. 
233      fp : array, optional 
234          The weighted sum of squared residuals of the spline approximation. 
235      ier : int, optional 
236          An integer flag about splrep success. Success is indicated if ``ier<=0``. 
237          If ``ier in [1,2,3]``, an error occurred but was not raised. Otherwise an 
238          error is raised. 
239      msg : str, optional 
240          A message corresponding to the integer flag, `ier`. 
241   
242      See Also 
243      -------- 
244      UnivariateSpline, BivariateSpline 
245      splprep, splev, sproot, spalde, splint 
246      bisplrep, bisplev 
247      BSpline 
248      make_interp_spline 
249   
250      Notes 
251      ----- 
252      See `splev` for evaluation of the spline and its derivatives. Uses the 
253      FORTRAN routine ``curfit`` from FITPACK. 
254   
255      The user is responsible for assuring that the values of `x` are unique. 
256      Otherwise, `splrep` will not return sensible results. 
257   
258      If provided, knots `t` must satisfy the Schoenberg-Whitney conditions, 
259      i.e., there must be a subset of data points ``x[j]`` such that 
260      ``t[j] < x[j] < t[j+k+1]``, for ``j=0, 1,...,n-k-2``. 
261   
262      This routine zero-pads the coefficients array ``c`` to have the same length 
263      as the array of knots ``t`` (the trailing ``k + 1`` coefficients are ignored 
264      by the evaluation routines, `splev` and `BSpline`.) This is in contrast with 
265      `splprep`, which does not zero-pad the coefficients. 
266   
267      The default boundary condition is 'not-a-knot', i.e. the first and second 
268      segment at a curve end are the same polynomial. More boundary conditions are 
269      available in `CubicSpline`. 
270   
271      References 
272      ---------- 
273      Based on algorithms described in [1]_, [2]_, [3]_, and [4]_: 
274   
275      .. [1] P. Dierckx, "An algorithm for smoothing, differentiation and 
276         integration of experimental data using spline functions", 
277         J.Comp.Appl.Maths 1 (1975) 165-184. 
278      .. [2] P. Dierckx, "A fast algorithm for smoothing data on a rectangular 
279         grid while using spline functions", SIAM J.Numer.Anal. 19 (1982) 
280         1286-1304. 
281      .. [3] P. Dierckx, "An improved algorithm for curve fitting with spline 
282         functions", report tw54, Dept. Computer Science,K.U. Leuven, 1981. 
283      .. [4] P. Dierckx, "Curve and surface fitting with splines", Monographs on 
284         Numerical Analysis, Oxford University Press, 1993. 
285   
286      Examples 
287      -------- 
288      You can interpolate 1-D points with a B-spline curve. 
289      Further examples are given in 
290      :ref:`in the tutorial <tutorial-interpolate_splXXX>`. 
291   
292      >>> import numpy as np 
293      >>> import matplotlib.pyplot as plt 
294      >>> from scipy.interpolate import splev, splrep 
295      >>> x = np.linspace(0, 10, 10) 
296      >>> y = np.sin(x) 
297      >>> spl = splrep(x, y) 
298      >>> x2 = np.linspace(0, 10, 200) 
299      >>> y2 = splev(x2, spl) 
300      >>> plt.plot(x, y, 'o', x2, y2) 
301      >>> plt.show() 
302   
303      """
304      res = _impl.splrep(x, y, w, xb, xe, k, task, s, t, full_output, per, quiet)
305      return res

308  def splev(x, tck, der=0, ext=0):
309      """ 
310      Evaluate a B-spline or its derivatives. 
311   
312      .. legacy:: function 
313   
314          Specifically, we recommend constructing a `BSpline` object and using 
315          its ``__call__`` method. 
316   
317      Given the knots and coefficients of a B-spline representation, evaluate 
318      the value of the smoothing polynomial and its derivatives. This is a 
319      wrapper around the FORTRAN routines splev and splder of FITPACK. 
320   
321      Parameters 
322      ---------- 
323      x : array_like 
324          An array of points at which to return the value of the smoothed 
325          spline or its derivatives. If `tck` was returned from `splprep`, 
326          then the parameter values, u should be given. 
327      tck : BSpline instance or tuple 
328          If a tuple, then it should be a sequence of length 3 returned by 
329          `splrep` or `splprep` containing the knots, coefficients, and degree 
330          of the spline. (Also see Notes.) 
331      der : int, optional 
332          The order of derivative of the spline to compute (must be less than 
333          or equal to k, the degree of the spline). 
334      ext : int, optional 
335          Controls the value returned for elements of ``x`` not in the 
336          interval defined by the knot sequence. 
337   
338          * if ext=0, return the extrapolated value. 
339          * if ext=1, return 0 
340          * if ext=2, raise a ValueError 
341          * if ext=3, return the boundary value. 
342   
343          The default value is 0. 
344   
345      Returns 
346      ------- 
347      y : ndarray or list of ndarrays 
348          An array of values representing the spline function evaluated at 
349          the points in `x`.  If `tck` was returned from `splprep`, then this 
350          is a list of arrays representing the curve in an N-D space. 
351   
352      See Also 
353      -------- 
354      splprep, splrep, sproot, spalde, splint 
355      bisplrep, bisplev 
356      BSpline 
357   
358      Notes 
359      ----- 
360      Manipulating the tck-tuples directly is not recommended. In new code, 
361      prefer using `BSpline` objects. 
362   
363      References 
364      ---------- 
365      .. [1] C. de Boor, "On calculating with b-splines", J. Approximation 
366          Theory, 6, p.50-62, 1972. 
367      .. [2] M. G. Cox, "The numerical evaluation of b-splines", J. Inst. Maths 
368          Applics, 10, p.134-149, 1972. 
369      .. [3] P. Dierckx, "Curve and surface fitting with splines", Monographs 
370          on Numerical Analysis, Oxford University Press, 1993. 
371   
372      Examples 
373      -------- 
374      Examples are given :ref:`in the tutorial <tutorial-interpolate_splXXX>`. 
375   
376      A comparison between `splev`, `splder` and `spalde` to compute the derivatives of a  
377      B-spline can be found in the `spalde` examples section. 
378   
379      """
380      if isinstance(tck, BSpline):
381          if tck.c.ndim > 1:
382              mesg = ("Calling splev() with BSpline objects with c.ndim > 1 is "
383                      "not allowed. Use BSpline.__call__(x) instead.")
384              raise ValueError(mesg)
385  
386          # remap the out-of-bounds behavior
387          try:
388              extrapolate = {0: True, }[ext]
389          except KeyError as e:
390              raise ValueError(f"Extrapolation mode {ext} is not supported "
391                               "by BSpline.") from e
392  
393          return tck(x, der, extrapolate=extrapolate)
394      else:
395          return _impl.splev(x, tck, der, ext)

398  def splint(a, b, tck, full_output=0):
399      """ 
400      Evaluate the definite integral of a B-spline between two given points. 
401   
402      .. legacy:: function 
403   
404          Specifically, we recommend constructing a `BSpline` object and using its 
405          ``integrate`` method. 
406   
407      Parameters 
408      ---------- 
409      a, b : float 
410          The end-points of the integration interval. 
411      tck : tuple or a BSpline instance 
412          If a tuple, then it should be a sequence of length 3, containing the 
413          vector of knots, the B-spline coefficients, and the degree of the 
414          spline (see `splev`). 
415      full_output : int, optional 
416          Non-zero to return optional output. 
417   
418      Returns 
419      ------- 
420      integral : float 
421          The resulting integral. 
422      wrk : ndarray 
423          An array containing the integrals of the normalized B-splines 
424          defined on the set of knots. 
425          (Only returned if `full_output` is non-zero) 
426   
427      See Also 
428      -------- 
429      splprep, splrep, sproot, spalde, splev 
430      bisplrep, bisplev 
431      BSpline 
432   
433      Notes 
434      ----- 
435      `splint` silently assumes that the spline function is zero outside the data 
436      interval (`a`, `b`). 
437   
438      Manipulating the tck-tuples directly is not recommended. In new code, 
439      prefer using the `BSpline` objects. 
440   
441      References 
442      ---------- 
443      .. [1] P.W. Gaffney, The calculation of indefinite integrals of b-splines", 
444          J. Inst. Maths Applics, 17, p.37-41, 1976. 
445      .. [2] P. Dierckx, "Curve and surface fitting with splines", Monographs 
446          on Numerical Analysis, Oxford University Press, 1993. 
447   
448      Examples 
449      -------- 
450      Examples are given :ref:`in the tutorial <tutorial-interpolate_splXXX>`. 
451   
452      """
453      if isinstance(tck, BSpline):
454          if tck.c.ndim > 1:
455              mesg = ("Calling splint() with BSpline objects with c.ndim > 1 is "
456                      "not allowed. Use BSpline.integrate() instead.")
457              raise ValueError(mesg)
458  
459          if full_output != 0:
460              mesg = (f"full_output = {full_output} is not supported. Proceeding as if "
461                      "full_output = 0")
462  
463          return tck.integrate(a, b, extrapolate=False)
464      else:
465          return _impl.splint(a, b, tck, full_output)

468  def sproot(tck, mest=10):
469      """ 
470      Find the roots of a cubic B-spline. 
471   
472      .. legacy:: function 
473   
474          Specifically, we recommend constructing a `BSpline` object and using the 
475          following pattern: `PPoly.from_spline(spl).roots()`. 
476   
477      Given the knots (>=8) and coefficients of a cubic B-spline return the 
478      roots of the spline. 
479   
480      Parameters 
481      ---------- 
482      tck : tuple or a BSpline object 
483          If a tuple, then it should be a sequence of length 3, containing the 
484          vector of knots, the B-spline coefficients, and the degree of the 
485          spline. 
486          The number of knots must be >= 8, and the degree must be 3. 
487          The knots must be a montonically increasing sequence. 
488      mest : int, optional 
489          An estimate of the number of zeros (Default is 10). 
490   
491      Returns 
492      ------- 
493      zeros : ndarray 
494          An array giving the roots of the spline. 
495   
496      See Also 
497      -------- 
498      splprep, splrep, splint, spalde, splev 
499      bisplrep, bisplev 
500      BSpline 
501   
502      Notes 
503      ----- 
504      Manipulating the tck-tuples directly is not recommended. In new code, 
505      prefer using the `BSpline` objects. 
506   
507      References 
508      ---------- 
509      .. [1] C. de Boor, "On calculating with b-splines", J. Approximation 
510          Theory, 6, p.50-62, 1972. 
511      .. [2] M. G. Cox, "The numerical evaluation of b-splines", J. Inst. Maths 
512          Applics, 10, p.134-149, 1972. 
513      .. [3] P. Dierckx, "Curve and surface fitting with splines", Monographs 
514          on Numerical Analysis, Oxford University Press, 1993. 
515   
516      Examples 
517      -------- 
518   
519      For some data, this method may miss a root. This happens when one of 
520      the spline knots (which FITPACK places automatically) happens to 
521      coincide with the true root. A workaround is to convert to `PPoly`, 
522      which uses a different root-finding algorithm. 
523   
524      For example, 
525   
526      >>> x = [1.96, 1.97, 1.98, 1.99, 2.00, 2.01, 2.02, 2.03, 2.04, 2.05] 
527      >>> y = [-6.365470e-03, -4.790580e-03, -3.204320e-03, -1.607270e-03, 
528      ...      4.440892e-16,  1.616930e-03,  3.243000e-03,  4.877670e-03, 
529      ...      6.520430e-03,  8.170770e-03] 
530      >>> from scipy.interpolate import splrep, sproot, PPoly 
531      >>> tck = splrep(x, y, s=0) 
532      >>> sproot(tck) 
533      array([], dtype=float64) 
534   
535      Converting to a PPoly object does find the roots at ``x=2``: 
536   
537      >>> ppoly = PPoly.from_spline(tck) 
538      >>> ppoly.roots(extrapolate=False) 
539      array([2.]) 
540   
541   
542      Further examples are given :ref:`in the tutorial 
543      <tutorial-interpolate_splXXX>`. 
544   
545      """
546      if isinstance(tck, BSpline):
547          if tck.c.ndim > 1:
548              mesg = ("Calling sproot() with BSpline objects with c.ndim > 1 is "
549                      "not allowed.")
550              raise ValueError(mesg)
551  
552          t, c, k = tck.tck
553  
554          # _impl.sproot expects the interpolation axis to be last, so roll it.
555          # NB: This transpose is a no-op if c is 1D.
556          sh = tuple(range(c.ndim))
557          c = c.transpose(sh[1:] + (0,))
558          return _impl.sproot((t, c, k), mest)
559      else:
560          return _impl.sproot(tck, mest)

563  def spalde(x, tck):
564      """ 
565      Evaluate a B-spline and all its derivatives at one point (or set of points) up 
566      to order k (the degree of the spline), being 0 the spline itself. 
567   
568      .. legacy:: function 
569   
570          Specifically, we recommend constructing a `BSpline` object and evaluate 
571          its derivative in a loop or a list comprehension. 
572   
573      Parameters 
574      ---------- 
575      x : array_like 
576          A point or a set of points at which to evaluate the derivatives. 
577          Note that ``t(k) <= x <= t(n-k+1)`` must hold for each `x`. 
578      tck : tuple 
579          A tuple (t,c,k) containing the vector of knots, 
580          the B-spline coefficients, and the degree of the spline whose  
581          derivatives to compute. 
582   
583      Returns 
584      ------- 
585      results : {ndarray, list of ndarrays} 
586          An array (or a list of arrays) containing all derivatives 
587          up to order k inclusive for each point `x`, being the first element the  
588          spline itself. 
589   
590      See Also 
591      -------- 
592      splprep, splrep, splint, sproot, splev, bisplrep, bisplev, 
593      UnivariateSpline, BivariateSpline 
594   
595      References 
596      ---------- 
597      .. [1] de Boor C : On calculating with b-splines, J. Approximation Theory 
598         6 (1972) 50-62. 
599      .. [2] Cox M.G. : The numerical evaluation of b-splines, J. Inst. Maths 
600         applics 10 (1972) 134-149. 
601      .. [3] Dierckx P. : Curve and surface fitting with splines, Monographs on 
602         Numerical Analysis, Oxford University Press, 1993. 
603   
604      Examples 
605      -------- 
606      To calculate the derivatives of a B-spline there are several aproaches.  
607      In this example, we will demonstrate that `spalde` is equivalent to 
608      calling `splev` and `splder`. 
609       
610      >>> import numpy as np 
611      >>> import matplotlib.pyplot as plt 
612      >>> from scipy.interpolate import BSpline, spalde, splder, splev 
613       
614      >>> # Store characteristic parameters of a B-spline 
615      >>> tck = ((-2, -2, -2, -2, -1, 0, 1, 2, 2, 2, 2),  # knots 
616      ...        (0, 0, 0, 6, 0, 0, 0),  # coefficients 
617      ...        3)  # degree (cubic) 
618      >>> # Instance a B-spline object 
619      >>> # `BSpline` objects are preferred, except for spalde() 
620      >>> bspl = BSpline(tck[0], tck[1], tck[2]) 
621      >>> # Generate extra points to get a smooth curve 
622      >>> x = np.linspace(min(tck[0]), max(tck[0]), 100) 
623       
624      Evaluate the curve and all derivatives 
625       
626      >>> # The order of derivative must be less or equal to k, the degree of the spline 
627      >>> # Method 1: spalde() 
628      >>> f1_y_bsplin = [spalde(i, tck)[0] for i in x ]  # The B-spline itself 
629      >>> f1_y_deriv1 = [spalde(i, tck)[1] for i in x ]  # 1st derivative 
630      >>> f1_y_deriv2 = [spalde(i, tck)[2] for i in x ]  # 2nd derivative 
631      >>> f1_y_deriv3 = [spalde(i, tck)[3] for i in x ]  # 3rd derivative 
632      >>> # You can reach the same result by using `splev`and `splder` 
633      >>> f2_y_deriv3 = splev(x, bspl, der=3) 
634      >>> f3_y_deriv3 = splder(bspl, n=3)(x) 
635       
636      >>> # Generate a figure with three axes for graphic comparison 
637      >>> fig, (ax1, ax2, ax3) = plt.subplots(1, 3, figsize=(16, 5)) 
638      >>> suptitle = fig.suptitle(f'Evaluate a B-spline and all derivatives') 
639      >>> # Plot B-spline and all derivatives using the three methods 
640      >>> orders = range(4) 
641      >>> linetypes = ['-', '--', '-.', ':'] 
642      >>> labels = ['B-Spline', '1st deriv.', '2nd deriv.', '3rd deriv.'] 
643      >>> functions = ['splev()', 'splder()', 'spalde()'] 
644      >>> for order, linetype, label in zip(orders, linetypes, labels): 
645      ...     ax1.plot(x, splev(x, bspl, der=order), linetype, label=label) 
646      ...     ax2.plot(x, splder(bspl, n=order)(x), linetype, label=label) 
647      ...     ax3.plot(x, [spalde(i, tck)[order] for i in x], linetype, label=label) 
648      >>> for ax, function in zip((ax1, ax2, ax3), functions): 
649      ...     ax.set_title(function) 
650      ...     ax.legend() 
651      >>> plt.tight_layout() 
652      >>> plt.show() 
653   
654      """
655      if isinstance(tck, BSpline):
656          raise TypeError("spalde does not accept BSpline instances.")
657      else:
658          return _impl.spalde(x, tck)

661  def insert(x, tck, m=1, per=0):
662      """ 
663      Insert knots into a B-spline. 
664   
665      .. legacy:: function 
666   
667          Specifically, we recommend constructing a `BSpline` object and using 
668          its ``insert_knot`` method. 
669   
670      Given the knots and coefficients of a B-spline representation, create a 
671      new B-spline with a knot inserted `m` times at point `x`. 
672      This is a wrapper around the FORTRAN routine insert of FITPACK. 
673   
674      Parameters 
675      ---------- 
676      x (u) : float 
677          A knot value at which to insert a new knot.  If `tck` was returned 
678          from ``splprep``, then the parameter values, u should be given. 
679      tck : a `BSpline` instance or a tuple 
680          If tuple, then it is expected to be a tuple (t,c,k) containing 
681          the vector of knots, the B-spline coefficients, and the degree of 
682          the spline. 
683      m : int, optional 
684          The number of times to insert the given knot (its multiplicity). 
685          Default is 1. 
686      per : int, optional 
687          If non-zero, the input spline is considered periodic. 
688   
689      Returns 
690      ------- 
691      BSpline instance or a tuple 
692          A new B-spline with knots t, coefficients c, and degree k. 
693          ``t(k+1) <= x <= t(n-k)``, where k is the degree of the spline. 
694          In case of a periodic spline (``per != 0``) there must be 
695          either at least k interior knots t(j) satisfying ``t(k+1)<t(j)<=x`` 
696          or at least k interior knots t(j) satisfying ``x<=t(j)<t(n-k)``. 
697          A tuple is returned iff the input argument `tck` is a tuple, otherwise 
698          a BSpline object is constructed and returned. 
699   
700      Notes 
701      ----- 
702      Based on algorithms from [1]_ and [2]_. 
703   
704      Manipulating the tck-tuples directly is not recommended. In new code, 
705      prefer using the `BSpline` objects, in particular `BSpline.insert_knot` 
706      method. 
707   
708      See Also 
709      -------- 
710      BSpline.insert_knot 
711   
712      References 
713      ---------- 
714      .. [1] W. Boehm, "Inserting new knots into b-spline curves.", 
715          Computer Aided Design, 12, p.199-201, 1980. 
716      .. [2] P. Dierckx, "Curve and surface fitting with splines, Monographs on 
717          Numerical Analysis", Oxford University Press, 1993. 
718   
719      Examples 
720      -------- 
721      You can insert knots into a B-spline. 
722   
723      >>> from scipy.interpolate import splrep, insert 
724      >>> import numpy as np 
725      >>> x = np.linspace(0, 10, 5) 
726      >>> y = np.sin(x) 
727      >>> tck = splrep(x, y) 
728      >>> tck[0] 
729      array([ 0.,  0.,  0.,  0.,  5., 10., 10., 10., 10.]) 
730   
731      A knot is inserted: 
732   
733      >>> tck_inserted = insert(3, tck) 
734      >>> tck_inserted[0] 
735      array([ 0.,  0.,  0.,  0.,  3.,  5., 10., 10., 10., 10.]) 
736   
737      Some knots are inserted: 
738   
739      >>> tck_inserted2 = insert(8, tck, m=3) 
740      >>> tck_inserted2[0] 
741      array([ 0.,  0.,  0.,  0.,  5.,  8.,  8.,  8., 10., 10., 10., 10.]) 
742   
743      """
744      if isinstance(tck, BSpline):
745  
746          t, c, k = tck.tck
747  
748          # FITPACK expects the interpolation axis to be last, so roll it over
749          # NB: if c array is 1D, transposes are no-ops
750          sh = tuple(range(c.ndim))
751          c = c.transpose(sh[1:] + (0,))
752          t_, c_, k_ = _impl.insert(x, (t, c, k), m, per)
753  
754          # and roll the last axis back
755          c_ = np.asarray(c_)
756          c_ = c_.transpose((sh[-1],) + sh[:-1])
757          return BSpline(t_, c_, k_)
758      else:
759          return _impl.insert(x, tck, m, per)

762  def splder(tck, n=1):
763      """ 
764      Compute the spline representation of the derivative of a given spline 
765   
766      .. legacy:: function 
767   
768          Specifically, we recommend constructing a `BSpline` object and using its 
769          ``derivative`` method. 
770   
771      Parameters 
772      ---------- 
773      tck : BSpline instance or tuple 
774          BSpline instance or a tuple (t,c,k) containing the vector of knots, 
775          the B-spline coefficients, and the degree of the spline whose  
776          derivative to compute 
777      n : int, optional 
778          Order of derivative to evaluate. Default: 1 
779   
780      Returns 
781      ------- 
782      `BSpline` instance or tuple 
783          Spline of order k2=k-n representing the derivative 
784          of the input spline. 
785          A tuple is returned if the input argument `tck` is a tuple, otherwise 
786          a BSpline object is constructed and returned. 
787   
788      See Also 
789      -------- 
790      splantider, splev, spalde 
791      BSpline 
792   
793      Notes 
794      ----- 
795   
796      .. versionadded:: 0.13.0 
797   
798      Examples 
799      -------- 
800      This can be used for finding maxima of a curve: 
801   
802      >>> from scipy.interpolate import splrep, splder, sproot 
803      >>> import numpy as np 
804      >>> x = np.linspace(0, 10, 70) 
805      >>> y = np.sin(x) 
806      >>> spl = splrep(x, y, k=4) 
807   
808      Now, differentiate the spline and find the zeros of the 
809      derivative. (NB: `sproot` only works for order 3 splines, so we 
810      fit an order 4 spline): 
811   
812      >>> dspl = splder(spl) 
813      >>> sproot(dspl) / np.pi 
814      array([ 0.50000001,  1.5       ,  2.49999998]) 
815   
816      This agrees well with roots :math:`\\pi/2 + n\\pi` of 
817      :math:`\\cos(x) = \\sin'(x)`. 
818   
819      A comparison between `splev`, `splder` and `spalde` to compute the derivatives of a  
820      B-spline can be found in the `spalde` examples section. 
821   
822      """
823      if isinstance(tck, BSpline):
824          return tck.derivative(n)
825      else:
826          return _impl.splder(tck, n)
827

829  def splantider(tck, n=1):
830      """ 
831      Compute the spline for the antiderivative (integral) of a given spline. 
832   
833      .. legacy:: function 
834   
835          Specifically, we recommend constructing a `BSpline` object and using its 
836          ``antiderivative`` method. 
837   
838      Parameters 
839      ---------- 
840      tck : BSpline instance or a tuple of (t, c, k) 
841          Spline whose antiderivative to compute 
842      n : int, optional 
843          Order of antiderivative to evaluate. Default: 1 
844   
845      Returns 
846      ------- 
847      BSpline instance or a tuple of (t2, c2, k2) 
848          Spline of order k2=k+n representing the antiderivative of the input 
849          spline. 
850          A tuple is returned iff the input argument `tck` is a tuple, otherwise 
851          a BSpline object is constructed and returned. 
852   
853      See Also 
854      -------- 
855      splder, splev, spalde 
856      BSpline 
857   
858      Notes 
859      ----- 
860      The `splder` function is the inverse operation of this function. 
861      Namely, ``splder(splantider(tck))`` is identical to `tck`, modulo 
862      rounding error. 
863   
864      .. versionadded:: 0.13.0 
865   
866      Examples 
867      -------- 
868      >>> from scipy.interpolate import splrep, splder, splantider, splev 
869      >>> import numpy as np 
870      >>> x = np.linspace(0, np.pi/2, 70) 
871      >>> y = 1 / np.sqrt(1 - 0.8*np.sin(x)**2) 
872      >>> spl = splrep(x, y) 
873   
874      The derivative is the inverse operation of the antiderivative, 
875      although some floating point error accumulates: 
876   
877      >>> splev(1.7, spl), splev(1.7, splder(splantider(spl))) 
878      (array(2.1565429877197317), array(2.1565429877201865)) 
879   
880      Antiderivative can be used to evaluate definite integrals: 
881   
882      >>> ispl = splantider(spl) 
883      >>> splev(np.pi/2, ispl) - splev(0, ispl) 
884      2.2572053588768486 
885   
886      This is indeed an approximation to the complete elliptic integral 
887      :math:`K(m) = \\int_0^{\\pi/2} [1 - m\\sin^2 x]^{-1/2} dx`: 
888   
889      >>> from scipy.special import ellipk 
890      >>> ellipk(0.8) 
891      2.2572053268208538 
892   
893      """
894      if isinstance(tck, BSpline):
895          return tck.antiderivative(n)
896      else:
897          return _impl.splantider(tck, n)
